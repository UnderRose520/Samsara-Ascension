# 天道规则系统 (Heavenly Dao Rules) — Implementation Plan

## Overview

天道规则是肉鸽核心机制。每个规则全局影响所有攻击（如火规则→所有攻击附带灼烧），规则可叠加，叠加越多难度越大、收益越大。双向影响：玩家获得强力效果，敌人也获得抗性+属性加成。

## 1. Data Model — CSV Design

**新文件**: `game/data/dao_rules/dao_rules.csv`

| Column | Type | Description |
|--------|------|-------------|
| `id` | string | Unique key (e.g. `fire_burn`, `ice_freeze`, `thunder_chain`, `kill_harvest`) |
| `name` | string | Display name (e.g. "火之天道", "杀伐之道") |
| `category` | string | `element` / `compound` / `mechanism` |
| `element` | string | Element key or empty (fire/ice/thunder/wood/earth/chaos/soul/empty) |
| `icon_path` | string | Asset path for icon (reserved slot) |
| `player_effect_kind` | string | Effect applied to player attacks (see effect DSL below) |
| `player_effect_value` | float | Magnitude of the player effect |
| `player_effect_param` | string | Extra param (e.g. status duration, chance) |
| `enemy_resist_element` | string | Element the enemies gain resistance to |
| `enemy_resist_value` | float | Enemy resistance multiplier (e.g. 0.7 = 30% less element damage) |
| `enemy_stat_mult_hp` | float | Enemy HP multiplier from this rule (e.g. 1.08) |
| `enemy_stat_mult_atk` | float | Enemy ATK multiplier |
| `enemy_stat_mult_speed` | float | Enemy speed multiplier |
| `reward_mult_gold` | float | Gold reward multiplier (e.g. 1.15) |
| `reward_mult_affix_quality` | int | Bonus quality tiers to affix offers |
| `tier` | int | 1=common, 2=rare, 3=legendary — controls acquisition frequency |
| `description` | string | Flavor text for UI |

### Effect DSL (player_effect_kind values)

| Kind | Description | Value | Param |
|------|-------------|-------|-------|
| `on_hit_status` | All attacks apply status | status_id | duration_sec |
| `on_hit_status_chance` | % chance to apply status | chance (0-1) | status_id:duration |
| `flat_damage_bonus` | Flat +N damage on all attacks | flat amount | — |
| `mult_damage` | All damage ×N | multiplier | — |
| `on_kill_heal` | Heal % HP on kill | heal_pct | — |
| `projectile_pierce` | +N pierce on all projectiles | pierce count | — |
| `on_dodge反击` | Perfect dodge deals damage | dodge_damage | radius |
| `lifesteal` | Lifesteal % of damage dealt | lifesteal_pct | — |
| `aoe_on_hit` | Each hit triggers AoE burst | aoe_damage | aoe_radius |

**Element Rules (7)**:
- 火之天道 — all attacks apply burn (3s, 100%)
- 冰之天道 — all attacks apply freeze (1.5s, 100%)

- 木之天道 — all attacks apply poison (5s, 100%)
- 混元之天道 — all attacks +15% damage, enemies -15% all resist
- 魂之天道 — all attacks +8% lifesteal
- 风之天道 — all attacks +1 pierce

**Compound Rules (5)**:

- 天劫之道 — every 10s, lightning strikes all enemies for 30% of player ATK
- 焚心之道 — all attacks apply burn + poison, enemies +20% fire resistance
- 破界之道 — all attacks ignore 30% enemy defense

**Mechanism Rules (5)**:
- 穿透之道 — all projectiles +3 pierce
- 暴击之道 — +25% crit rate, +30% crit damage

## 3. Application Points

### 3a. Player Attack Effects — AffixHolder Integration

**修改文件**: `game/systems/affix/affix_holder.gd`

In `proc_on_hit(target, damage_ctx)`:
- After processing existing on_hit effects, iterate `RunContext.active_dao_rules`
- For each rule with `player_effect_kind == "on_hit_status"`: call `target.get_node("StatusComponent").apply_status(status, duration)`
- For `"on_hit_status_chance"`: roll chance, then apply
- For `"aoe_on_hit"`: emit EventBus signal for AoE burst

In `_recompute()` or `build_damage_context()`:
- For rules with `"flat_damage_bonus"`: add to flat_attack accumulator
- For `"mult_damage"`: add to appropriate bucket
- For `"projectile_pierce"`: add to projectile_pierce counter
- For `"lifesteal"`: add to lifesteal accumulator

### 3b. Enemy Resistance — DamagePipeline Integration

**修改文件**: `game/systems/combat/damage_pipeline.gd`

In `compute_pve(ctx)`:
- After computing base damage, before applying buckets:
  - Check if `ctx.element` matches any active rule's `enemy_resist_element`
  - If so, multiply damage by `RunContext.get_dao_rule_enemy_resist(element)`

### 3c. Enemy Stat Scaling — Enemy Spawn Integration

**修改文件**: `game/systems/combat/enemy_spawn_registry.gd`

In `roll_instance_stats()` or in `ArenaBase._spawn_enemy_dummy()`:
- After rolling base stats, apply `RunContext.get_dao_rule_enemy_hp_mult()` to HP
- Apply `RunContext.get_dao_rule_enemy_atk_mult()` to ATK
- Apply `RunContext.get_dao_rule_enemy_speed_mult()` to speed

### 3d. On-Kill Effects — RunController Integration

**修改文件**: `game/scenes/rooms/run_controller.gd`

In the enemy_killed handler:
- Iterate `RunContext.active_dao_rules`
- For rules with `player_effect_kind == "on_kill_heal"`: heal player
- For `"连锁之道"`: find nearby enemies, deal chain damage

### 3e. Periodic Effects — Player or HordeController

For rules like "天劫之道" (periodic lightning):
- Add a timer in the player script or a dedicated DaoRuleTicker node
- Every N seconds, apply the periodic effect

## 4. Difficulty Scaling

The `dao_rule_difficulty_mult` is computed as:

```
per_rule_mult = 1.0 + (active_rules_count - 1) * 0.06  # 6% per extra rule
dao_rule_difficulty_mult = product of all enemy_stat_mult values
```

Combined in `get_enemy_hp_mult()`:
```
return DaoHeartConfig.enemy_hp_mult(dao_heart, is_boss)
    * heart_demon_mult
    * dao_rule_difficulty_mult
```

## 5. Reward Scaling

```
dao_rule_reward_mult = 1.0 + active_rules_count * 0.08  # 8% gold per rule
dao_rule_quality_bonus = floor(active_rules_count / 2)  # +1 quality tier per 2 rules
```

Applied in:
- `RunController` gold reward calculation: `gold_earned * dao_rule_reward_mult`
- `AffixOfferSelector.roll_offers()`: shift quality weights by `dao_rule_quality_bonus`

## 6. Acquisition — When Rules Appear

**修改文件**: `game/systems/affix/affix_offer_selector.gd`

Add a new function `roll_dao_rule_offer()`:
- After each room clear (combat/event), alongside the existing affix 3-pick-1
- 25% base chance to also offer a 天道规则 choice (3-pick-1 of rules)
- Chance increases by 5% per stage (stage 1: 25%, stage 2: 30%, ... stage 5: 45%)
- Already-active rules are excluded from the pool
- Higher-tier rules appear more frequently in later stages
- First rule is always offered after room 2 (guaranteed early hook)

**修改文件**: `game/scenes/rooms/run_controller.gd`

In `_post_room_rewards()`:
- After affix choice flow, check `roll_dao_rule_offer()`
- If triggered, show DaoRuleChoicePanel before path choice
- Store chosen rule via `RunContext.add_dao_rule()`
