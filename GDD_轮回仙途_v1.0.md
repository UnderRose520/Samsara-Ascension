## 1. 游戏概述

### 1.1 一句话描述

> 一款修仙题材的混合型 Roguelite 游戏，玩家扮演轮回转世的修士，
> 每一局经历从炼气到飞升的完整修仙人生，通过功法搭配、天象利用、
> 因果抉择，打造独一无二的修仙 Build，最终渡劫飞升或含恨陨落。

### 1.2 核心体验

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  动作层 —— 操控施法、闪避腾挪、法术连招（手感爽）    │
│  构筑层 —— 功法搭配、词条联动、Build 成型（策略爽）  │
│  表现层 —— 剑光满屏、雷劫降世、万法归一（视觉爽）    │
│                                                     │
│  三层体验在同一个战斗中同时发生                       │
│  不是三个系统拼接，而是一个完整体验                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 1.3 目标受众

- **主要受众：** 喜欢肉鸽游戏的核心玩家（Hades / 杀戮尖塔 / 以撒的结合玩家）
- **次要受众：** 修仙/仙侠文化爱好者
- **潜在受众：** 喜欢 Build 构筑和数值爆炸的休闲玩家

### 1.4 竞品分析

| 竞品 | 我们学习的 | 我们不同的 |
|------|-----------|-----------|
| Hades | 打击感、叙事与肉鸽结合 | 我们用天象系统替代线性叙事 |
| 杀戮尖塔 | 词条联动深度、路径选择 | 我们是动作而非卡牌 |
| 以撒的结合 | 道具池广度、组合爆炸 | 我们有修仙主题和领悟系统 |
| 暖雪 | 修仙美术风格 | 我们有天象系统和因果系统 |
| 吸血鬼幸存者 | 弹幕密度、数值膨胀感 | 我们有主动操作和策略深度 |

## 2. 核心理念与设计哲学

### 2.1 三个心理钩子

每一项设计都必须服务以下至少一个钩子：

```
钩子 1：可能性的诱惑
  "这局说不定能抽到那个逆天功法……"
  → 随机奖励、隐藏内容、小概率事件

钩子 2：差一点的遗憾
  "就差一个词条就凑成完美 combo 了！"
  → 给玩家看到"接近完成"的构筑，让他不甘心

钩子 3：下次会更好的掌控感
  "我知道该怎么选了，再来一局肯定更强"
  → 局外成长 + 玩家对系统的理解加深
```

### 2.2 五项设计原则

```
1. 选择大于运气
   → 所有随机都给玩家"选择权"（三选一而非纯掷骰）
   → 运气决定"你有什么选项"，选择决定"你走什么路"

2. 发现大于告知
   → 词条联动不直接告诉玩家，让玩家自己发现 combo
   → 首次发现隐藏交互时弹出"领悟"提示（惊喜感）

3. 节奏大于强度
   → 不能一直爽，要有弱→强→碾压的节奏
   → 也不能一直弱，每个阶段都要有"进展感"

4. 死亡也是收获
   → 每次死亡都有轮回点数、图鉴更新、解锁进度
   → 死亡界面要让玩家觉得"值了"而非"亏了"

5. 修仙即肉鸽
   → 轮回 = 重开局
   → 境界 = 关卡进度
   → 功法 = 词条
   → 渡劫 = Boss 战
   → 飞升 = 通关
   → 所有机制都有修仙叙事包装
```

### 2.3 核心循环

```
一局循环（25-40分钟）：

  选择灵根/角色
      ↓
  进入修炼（战斗房间）
      ↓
  ┌─→ 击败敌人 → 获得功法/词条选择 ─┐
  │                                    ↓
  │              进入下一个房间（战斗/事件/商店）
  │                                    ↓
  │              Boss 战 → 境界突破 ──┤
  │                                    ↓
  └────────────────────────────────────┘
      ↓
  渡劫飞升（最终 Boss）
      ↓
  结算 → 轮回点数 → 局外升级 → 再来一局

每次循环之间的"上瘾点"：
  · "这局的 Build 差一点就完美了"
  · "刚解锁了新功法，下一局可以试试"
  · "还差 200 点就解锁新角色了"
```

│  暂停             │  Esc           │  暂停菜单      │
└──────────────────────────────────────────────────┘
```

### 3.2 移动系统

```
基础移动速度：  300 像素/秒
加速状态：      400 像素/秒（持续1秒，触发条件待定）
减速状态：      200 像素/秒（受控效果时）

惯性系统：
  · 松开按键后有 0.05 秒的微滑行（手感更流畅）
  · 不做过度惯性（避免操作延迟感）

碰撞：
  · 角色碰撞体积为圆形，半径 12 像素
  · 与墙壁碰撞后停止（不反弹）
  · 与敌人碰撞时产生轻微推挤力
```

### 3.3 闪避系统（御风步）

```
┌──────────────────────────────────────────────────┐
│  闪避参数                                        │
│                                                  │
│  冲刺距离：      150 像素                         │
│  冲刺时间：      0.2 秒                          │
│  无敌时间：      0.25 秒（覆盖冲刺全程 + 余量）   │
│  冷却时间：      1.0 秒                          │
│  冲刺方向：      当前移动方向 / 鼠标方向（可选）   │
│                                                  │
│  视觉效果：                                       │
│  · 冲刺时角色变为半透明残影                       │
│  · 留下一道灵气轨迹（颜色对应灵根属性）           │
│  · 无敌帧期间角色闪烁                            │
│                                                  │
│  可被词条修改：                                    │
│  · 冷却时间可缩短                                 │
│  · 距离可增加                                     │
│  · 可附加伤害/效果（如雷系闪避放电）              │
│  · 可增加闪避次数（多段闪避）                     │
└──────────────────────────────────────────────────┘
```

## 5. 修炼境界系统

  ├────────┼────────┼───────────┼──────────┼───────────────────┤
  │ 炼气期  │ 第1关   │ 3 → 5    │ 初入仙途  │ 弱，但充满可能性   │
  │ 筑基期  │ 第2关   │ 5 → 7    │ 法力初成  │ combo 开始成型     │
  │ 金丹期  │ 第3关   │ 7 → 9    │ 质变突破  │ 核心 build 完成    │
  │ 元婴期  │ 第4关   │ 9 → 12   │ 大能降临  │ 构筑深度展开       │
  │ 渡劫期  │ 最终关  │ 12        │ 天劫降临  │ 全部实力的终极检验  │
  │ 飞升    │ 通关    │ —         │ 得道成仙  │ 游戏结束（本次轮回）│
  └────────┴────────┴───────────┴──────────┴───────────────────┘

  │  · 角色被灵气光柱包裹                                  │
  │  · 屏幕轻微震动                                       │
  │  · 词条槽扩展动画（新槽位亮起）                        │

### 6.2 伤害计算流水线

```
伤害计算分为 8 个阶段，按顺序执行：

  阶段 1：基础伤害
    base_damage = 攻击力 × 技能倍率

  阶段 2：属性加成
    base_damage ×= (1 + 对应属性加成%)
    // 如火灵根+15%，烈日天气+30%，火之亲和+20%

  阶段 3：词条加成（加法叠加）
    base_damage += 所有"伤害 +X"类词条的总和

  阶段 4：词条乘算
    base_damage ×= 所有"伤害 ×倍率"类词条的乘积
    // 如：焚天(×1.0忽略护甲)、纯阳真火(×1.3)

  阶段 5：暴击判定
    if random() < 暴击率:
      base_damage ×= 暴击倍率（默认 1.5）
      标记为暴击

  阶段 6：防御减免
    final_damage = base_damage × (100 / (100 + 敌人防御))
    // 如果有穿甲效果，在此阶段降低敌人防御

  阶段 7：伤害修正
    final_damage ×= 境界差距修正（高境界 +50%，低境界 -30%）
    final_damage ×= Boss 特殊修正（如有）

  阶段 8：最终取整
    final_damage = max(1, floor(final_damage))

  阶段 9：触发后处理
    → 触发"命中时"类词条
    → 触发"暴击时"类词条（如果是暴击）
    → 触发"击杀"类词条（如果目标死亡）
    → 更新连击计数器
    → 显示伤害数字
```

## 7. 天象系统

### 7.1 设计理念

```

## 11. 随机事件系统

### 11.1 事件设计原则

```
好的随机事件 = 叙事背景 + 有意义的选择 + 不确定的结果 + 后续影响

  · 标题和描述文本（修仙风格叙事）
  · 2-4 个选项
  · 每个选项的概率结果
  · 因果标记（可能影响后续事件）
  · 稀有度（决定出现频率）

## 12. 因果系统

### 12.1 设计理念

```
因果系统 = 选择的长期后果

在肉鸽中加入因果系统，让选择不只是"当下最优解"，还要考虑"后续影响"。
这在肉鸽中很罕见，是我们的核心差异化之一。

因果不是惩罚系统，而是"选择有重量"的叙事系统。
恶行不一定是坏的（可能带来强力效果），善行不一定是好的（可能错过机遇）。
```

### 12.2 因果标记类型

```
┌──────────┬──────────────────────────────────────────────┐
│  标记     │  获得方式                                     │
├──────────┼──────────────────────────────────────────────┤
│  善缘     │  帮助 NPC、牺牲自己的利益救人                  │
│  恶行     │  抢夺 NPC、见死不救                           │
│  贪念     │  选择高风险高回报的选项                       │
│  道心     │  选择符合自己路线的坚定选项                   │
│  执念     │  同一类型的事件重复选择相同方向                │
│  逆天     │  选择挑战不可能的选项（如对仙人出手）          │
└──────────┴──────────────────────────────────────────────┘

累积效果：
  · 每个标记独立计数
  · 累积到 3 个同类标记触发对应"因果事件"
  · 标记之间可能互相影响

因果事件对照表：

  善缘 ×3  → "福报"事件：免费获得仙品词条
  恶行 ×3  → "心魔"事件：强制精英战，胜利后获强力 buff
  贪念 ×3  → "天谴"事件：损失 30% 生命，但获得天品词条
  道心 ×3  → "顿悟"事件：任选一个功法领悟 +2 层
  执念 ×3  → "执迷"事件：你最常用的词条效果 ×2，但其他词条 -20%
  逆天 ×3  → "天劫提前"事件：额外 Boss 战，胜利后境界跳一级
```

### 12.3 因果系统与心魔事件联动

```
当因果标记总数 ≥ 3（不限类型）时：
  → 心魔试炼事件可触发（见 11.3 稀有事件 04）

心魔试炼是因果系统的核心奖励机制：
  · 击败心魔后所有因果标记清除
  · 获得"双词条槽"（极强的永久奖励）
  · 这让因果系统从"需要管理的负担"变成"值得追求的目标"
  · 玩家会主动选择积累因果，以触发心魔试炼

设计意图：
  · 新手：尽量避免因果（不理解系统）
  · 进阶：理解因果后选择性积累
  · 高手：主动规划因果积累路线，精准触发心魔
```

## 14. Boss 设计

### 14.1 Boss 通用规则

```
所有 Boss 共有机制：
  · 多阶段战斗（2-4 阶段，每阶段不同行为模式）
  · 阶段转换时有短暂过场（3-5 秒无敌 + 动画）
  · 弱点暴露机制（特定攻击后有 2-3 秒弱点窗口）
  · 狂暴机制（血量低于 20% 时攻击速度和伤害提升）
  · 击杀后有慢动作特写 + 境界突破动画
```

```

## 17. 动态难度系统

### 17.1 设计理念

```
不是"简单模式"，而是让强者更刺激、弱者不绝望的隐形调节。

玩家永远不应该感知到难度在变化。
目标是让每一局都"刚好在你能力边缘"——不太简单（无聊），不太难（沮丧）。
```

### 17.2 构筑完成度评估

```
系统实时评估玩家的"构筑完成度"：

  构筑完成度 = 以下因素加权平均：
  
    词条数量占比       权重 25%    （拥有词条数 / 该阶段预期词条数）
    词条品质均值       权重 20%    （平均品质 / 该阶段预期品质）
    功法领悟层数       权重 25%    （平均领悟层数 / 最大层数）
    combo 联动完成度   权重 20%    （已触发联动 / 可触发联动）
    属性值对比         权重 10%    （当前属性 / 该阶段预期属性）

  结果：
    完成度 > 80%  → 构筑很强
    完成度 40-80% → 正常
    完成度 < 40%  → 构筑偏弱
```

### 17.3 动态调节项

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  构筑很强（完成度 > 80%）                                  │
│  · 敌人生命 +10%，攻击 +5%                                │
│  · 精英出现率 +15%（给你施展的机会）                       │
│  · 奖励关概率 +5%（奖励强者）                             │
│  · Boss 多使用 1 个技能模式                               │
│  · 掉落品质小幅提升（保持正反馈）                          │
│                                                          │
│  构筑偏弱（完成度 < 40%）                                  │
│  · 敌人生命 -10%，攻击 -5%                                │
│  · 精英出现率 -10%                                        │
│  · 商店出现更多该玩家已选属性的词条（引导性）              │
│  · 事件中"安全"路径的收益提高                             │
│  · Boss 血量 -10%                                        │
│  · 隐藏房出现率 +5%（补偿机会）                           │
│                                                          │
│  注意：                                                   │
│  · 调节幅度始终很小（±10-15%），玩家不会感知              │
│  · 不影响词条掉落品质（保持随机公平性）                    │
│  · 不影响成就/解锁条件                                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
