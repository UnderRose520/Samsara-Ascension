## 1. 游戏概述

### 1.1 一句话描述

> 一款修仙题材的混合型 Roguelite 游戏，玩家扮演轮回转世的修士，每一局经历从炼气到飞升的完整修仙人生。通过功法搭配、天象利用、地形交互、因果抉择、灵宠并肩、器灵觉醒，打造独一无二的修仙 Build，最终渡劫飞升——或在无尽深渊中挑战极限。

### 1.2 核心体验

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  动作层 —— 操控施法、闪避腾挪、法术连招（手感爽）        │
│  构筑层 —— 功法搭配、词条联动、Build 成型（策略爽）      │
│  表现层 —— 剑光满屏、雷劫降世、万法归一（视觉爽）        │
│  环境层 —— 天象变化、地形交互、借势而行（策略+操作爽）    │
│  伙伴层 —— 灵宠协同、器灵觉醒、心意相通（陪伴感）        │
│                                                         │
│  五层体验在同一个战斗中同时发生                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 核心卖点（差异化标签）

```
宣传核心标签：

  "天地为棋，万法归一"

  · 天象不是背景，是你的武器
    → 积水导雷、干燥引爆、冰面滑行、毒雾融合
    → 每一局的天象都在改变你的战斗方式

  · 词条不是堆叠，是化学反应
    → 灼烧 × 引燃 × 爆燃 × 火借风势 = 一发子弹烧全场
    → 50+ 种联动组合，玩家自己发现

  · 灵宠不是摆设，是战友
    → V+方向键触发协同技能
    → 火灵雀冲锋留火带、雷灵貂瞬移雷击、土灵龟为你挡刀

  · 死亡不是结束，是下一世的开始
    → 前世遗泽引导方向、轮回点数永久成长、因果系统延续

  · 每局都是不同的修仙人生
    → 天命种子暗中影响机缘走向、身世特质赋予独特起点
    → 同样的灵根，走出完全不同的道路

  · 道统不是彩蛋，是修炼的里程碑
    → 凑齐特定词条组合，觉醒"道统"——称号、特效、强化三位一体
    → 每个角色有专属道统，解锁终极飞升结局
```

### 1.4 目标受众

- **主要：** 肉鸽核心玩家（Hades / 杀戮尖塔 / 以撒的结合）
- **次要：** 修仙/仙侠文化爱好者
- **潜在：** 喜欢 Build 构筑和数值爆炸的休闲玩家
- **长期：** 无尽模式排行榜硬核玩家

### 1.5 竞品分析

| 竞品 | 学习的 | 不同的 |
|------|--------|--------|
| Hades | 打击感、叙事融合、斩杀特写 | 天象地形交互、因果系统、道统构筑 |
| 杀戮尖塔 | 词条联动深度 | 动作而非卡牌，灵宠/器灵 |
| 以撒的结合 | 道具池组合爆炸 | 修仙主题、功法领悟 |
| 暖雪 | 修仙美术风格 | 天象系统、灵宠协同、无尽模式 |
| 吸血鬼幸存者 | 弹幕密度、数值膨胀 | 主动操作和策略深度 |
| 修仙模拟器 | 天命/出身随机开局 | Roguelite 节奏而非模拟经营 |

## 2. 核心理念与设计哲学

### 2.1 四个心理钩子

```
钩子 1：可能性的诱惑
  "这局说不定能抽到那个逆天功法……"
  → 随机奖励、隐藏内容、小概率事件、灵宠/器灵

钩子 2：差一点的遗憾
  "就差一个词条就凑成完美 combo 了！"
  → Combo 指示器让玩家看到"完成度 75%"的 build

钩子 3：下次会更好的掌控感
  "我知道该怎么选了，再来一局肯定更强"
  → 前世遗泽 + 局外成长 + 系统理解加深

钩子 4：无尽的攀登
  "我还能更强……深渊第 50 层见"
  → 无尽模式的排行榜和极限挑战
```

### 2.2 七项设计原则

```
1. 选择大于运气
   → 所有随机都给玩家"选择权"
   → 难度让玩家自己选（开局选择 + 心魔试炼）

2. 发现大于告知
   → 词条联动不直接告知，让玩家自己发现 combo
   → 天象 × 地形 × 词条的交互等待玩家探索
   → 首次天象交互通过场景演出引导，而非弹窗教程

3. 节奏大于强度
   → 不能一直爽，要有弱→强→碾压的节奏
   → Boss 门前有"呼吸点"，战斗内有高潮低潮
   → 天机一击特写不打断节奏（0.3-1秒慢动作内完成）

4. 死亡也是收获
   → 前世遗泽引导下一局方向
   → 轮回点数永久成长
   → 名场面回顾让死亡有记忆点

5. 修仙即肉鸽
   → 轮回/境界/功法/渡劫/飞升 全部对应肉鸽机制

6. 每一秒都有意义
   → 战斗结束直接选路径（无跑图）
   → 过渡动画同时传递信息
   → Boss 门前暂停是"有意义的调息"

7. 透明的掌控感
   → 难度玩家自己选（开局 + 心魔试炼）
   → Build 可视化五层体系（含道统里程碑）
   → 灵宠/器灵成长全程可见
   → 因果标记 HUD 可见（但触发结果仍需探索）
   → 道韵衰减机制可在伤害面板查看当前衰减状态
```

### 2.3 核心循环

```
一局循环（15-30分钟·主线 / 无上限·无尽）：

  选择难度 → 选择灵根/角色 → 天命揭示 → 前世遗泽（如非首局）
      ↓
  进入修炼（战斗房间）
      ↓
  ┌─→ 击败敌人 → 天机一击特写 → 直接选择下一路径 → 灵宠/器灵成长 ─┐
  │                                                                ↓
  │          进入下一个房间（战斗/事件/商店/奖励关）               ↓
  │          → 天象变化时场景内交互演示（首次触发）                ↓
  │                                                                ↓
  │          Boss 门前调息 → Boss 战 → 道统检测 → 境界突破 ────┤
  │                                                                ↓
  └────────────────────────────────────────────────────────────────┘
      ↓
  主线：渡劫飞升 → 结算 → 角色记忆碎片 → 轮回点数 → 局外升级
  无尽：深渊层 → 结算 → 排名 → 轮回点数
      ↓
  再来一局
```

## 3. 难度系统

### 3.1 设计理念

```
难度系统的核心矛盾：
  · 新手需要低门槛进入 → 不能一上来就难
  · 老手需要挑战感     → 不能一直简单
  · 玩家讨厌"被看低"   → 不能用"简单/普通/困难"这种标签
  · 玩家喜欢掌控感     → 让玩家自己选，而非系统暗改

解决方案：用修仙叙事包装难度选择

  不是"选难度"
  而是"选择你的道心"
```

### 3.4 难度选择的心理设计

```
关键原则：不要让玩家觉得"选简单 = 丢人"

  1. 没有"简单/普通/困难"这种直白标签
     → 用修仙术语包装："问道/悟道/证道"
     → 每个都有正面描述，没有贬义

  2. 每个难度都有独特价值
     → 问道：适合体验剧情和学习系统
     → 悟道：标准修仙体验
     → 证道：有额外奖励和称号
     → 不是"弱者选简单"，而是"不同道心不同路"

  3. 可以随时修改
     → 设置里随时切换（下一层生效）
     → 不会被"锁定"在某个难度
     → 鼓励玩家"先问道入门，再证道挑战"

  4. 无尽模式难度独立
     → 无尽模式不使用道心系统
     → 而是用层数自然递增难度
     → 硬核玩家的挑战在无尽模式中

  5. 首次游戏推荐问道
     · 第一次进入游戏时，默认选中"问道"
     · 下方提示："推荐首次游玩选择此道心"
     · 但不强制，玩家可以自由选择
     · 选择证道时额外提示："此道心适合有经验的修士"

  6. 道心选择后可查看完整影响
     · 选中某个道心后，展开"详细影响"面板
     · 列出所有数值变化
     · 信息完全透明
```

### 3.5 心魔试炼（游戏内难度调节）

```
心魔试炼不是每关必出，而是事件型可选挑战：

  触发条件：
    · 问道难度：不触发（新手保护）
    · 悟道难度：每关 20% 概率出现在事件房
    · 证道难度：每关必出 1 次

  ┌──────────────────────────────────────────────┐
  │                                              │
  │  房间中央矗立着一面古老的铜镜。              │
  │  镜中映出你的身影，但眼神不同……              │
  │                                              │
  │  ┌──────────────────────────────────────┐   │
  │  │                                      │   │
  │  │  选项 A：接受心魔试炼                 │   │
  │  │  → 本层剩余房间敌人全属性 +15%        │   │
  │  │  → 但本层所有词条品质 +1 阶           │   │
  │  │  → 击败 Boss 后获得心魔碎片           │   │
  │  │  → 获得因果标记"逆天"                 │   │
  │  │                                      │   │
  │  │  选项 B：参悟铜镜                     │   │
  │  │  → 消耗 30 灵石                      │   │
  │  │  → 获得 1 个随机词条（正常品质）      │   │
  │  │                                      │   │
  │  │  选项 C：离开                         │   │
  │  │  → 什么都不发生                       │   │
  │  │                                      │   │
  │  └──────────────────────────────────────┘   │
  │                                              │
  └──────────────────────────────────────────────┘

  心魔碎片用途：
    · 收集 3 个 → 下一局可选择"心魔强化"开局
    · 心魔强化：开局获得 1 个仙品词条，但敌人 +10%
    · 是追求极限的玩家的额外挑战+奖励通道
```

### 3.6 无尽模式难度（自然递增 + 显式选择）

```
无尽模式不使用道心系统，用自然递增 + 显式三档选择：

  自然递增（基础）：
    敌人每层 +15%生命 +10%攻击 +5%防御

  每 10 层前可选择：
    ┌──────────────────────────────────────────────┐
    │                                              │
    │  你来到了深渊的第 20 层……                     │
    │  前方的灵气波动更加强烈。                     │
    │                                              │
    │  [挑战模式] 敌人+20%血，每层额外掉1词条      │
    │  [标准模式] 正常难度                         │
    │  [沉稳模式] 敌人-10%血，排名不计入主榜       │
    │                                              │
    └──────────────────────────────────────────────┘

  · 挑战模式：追求更高排名的玩家
  · 标准模式：正常体验
  · 沉稳模式：练习/休闲，但排名有代价（不计入主榜）
```

### 7.2 伤害流水线（9阶段 + 道韵衰减）

```
完整伤害计算流程：

  1.基础伤害 → 2.属性加成 → 3.词条加算
  → 4.词条乘算（经道韵衰减修正）→ 5.环境乘算（经道韵衰减修正）
  → 6.灵宠/器灵加成（经道韵衰减修正）→ 7.暴击
  → 8.防御减免 → 9.修正
  → 10.后处理（触发效果+天机一击判定+显示数字）

道韵衰减机制：

  设计意图：
    防止多个乘算来源线性叠加导致伤害指数爆炸，同时
    保留"每多凑一个词条都有收益"的构筑满足感。
    采用递减收益（而非硬上限），玩家仍可自由追求极限。

  伤害桶分类（4个桶）：
    桶A·词条乘算：联动词条效果（纯阳真火、焚天、五行归元等）
    桶B·环境乘算：天象加成、地形加成
    桶C·伙伴乘算：灵宠被动加成、器灵加成
    桶D·境界乘算：境界天赋、局外成长加成

  同桶内叠加规则：
    第1个乘算来源：100% 效果
    第2个乘算来源：80% 效果（即超出基准的部分×0.8）
    第3个及以上：  60% 效果（即超出基准的部分×0.6）

  计算示例：
    桶A有2个乘算来源：纯阳真火(×1.3) + 焚天(×1.5)
      纯阳真火：×1.3（第1个，100%效果）
      焚天：1 + (1.5-1) × 0.8 = ×1.4（第2个，80%效果）
      桶A总乘算：×1.3 × 1.4 = ×1.82

    vs 无递减：×1.3 × 1.5 = ×1.95
    差异7%，感知不明显

    如果桶A有第3个乘算(×1.3)：
      有递减：1 + (1.3-1) × 0.6 = ×1.18 → 桶A总计 ×1.3 × 1.4 × 1.18 = ×2.15
      无递减：×1.3 × 1.5 × 1.3 = ×2.54
      差异15%，有效防止溢出

    最终伤害 = 基础伤害 × 桶A × 桶B × 桶C × 桶D × 暴击

  设计效果：
    · 桶间相乘（鼓励跨系统构筑：词条+天象+灵宠+境界）
    · 同桶递减（防止纯堆单一来源的极端数值）
    · 3个桶各叠2个乘算 > 1个桶叠6个乘算
    · 自然引导玩家关注天象利用和灵宠搭配（跨桶收益更高）
    · 每多拿一个词条仍有收益（不设硬上限），但边际递减
```

## 8. 天象与地形交互系统

### 8.1 设计理念

```
天象不是数值 buff，而是改变战斗环境的物理力量。

核心原则：

## 12. 灵宠系统

### 12.1 灵宠概览

```
灵宠 = 跟随战斗的伙伴
  · 有独立生命、技能、成长
  · 通过协同技能系统与玩家配合
  · 与词条/天象/器灵深度联动
  · 同时出战1只，最多收藏3只
```

### 12.2 灵宠获取

```
  稀有事件（天降异宝蛋）/ 灵兽巢穴事件
  Boss掉落15% / 隐藏房10% / 坊市购买
  天象/因果专属事件 / 无尽每20层奖励
```

### 12.3 灵宠获取保底机制

```
设计理念：
  灵宠是核心体验伙伴层的支柱（见1.2），不应出现连续多局
  无法获取灵宠的情况。保底机制隐性运行，玩家不直接感知。

保底规则：
  · 前3关（炼气→金丹）中，至少有 1 次灵宠获取机会触发
    （可以是事件房出现灵兽巢穴、Boss掉落、隐藏房掉落，
      任一触发即满足保底）
  · 若到第3关Boss战结束仍未获得任何灵宠，则Boss战后
    必定出现"灵兽巢穴"事件（从火灵雀/水灵蛟/雷灵貂/
    木灵蝶/土灵龟 5种基础灵宠中选择1只）
  · 保底不告知玩家（保持"发现大于告知"原则），但确保
    玩家在前3关内至少有1只灵宠

前世遗泽联动：
  · 若上一局拥有灵宠但角色死亡，前世遗泽中增加选项：
    "灵宠魂魄" → 开局获得随机基础灵宠蛋（见20.2章）
```

### 12.5 灵宠协同系统（核心交互）

```

  · 你始终控制你的角色（不切换控制）
  · 灵宠自动战斗（现有系统）
  · V+方向键触发"协同动作"（15-30秒冷却）
  · 每种灵宠的协同动作不同
  · 操作简单，不打断战斗节奏

操作方式：
  V + W（前）：灵宠冲锋 — 冲向鼠标方向，造成伤害+击退
  V + S（后）：灵宠护卫 — 环绕你，5秒内为你挡弹幕
  V + A（左）：灵宠探路 — 灵宠前冲，照亮迷雾/提示隐藏
  V + D（右）：灵宠爆发 — 灵宠释放强化版技能，之后10秒虚弱

每种灵宠的具体协同动作：

  火灵雀：
    V+W → 火焰冲锋（冲向目标，路径留火焰带3秒）
    V+S → 烈焰守护（环绕你释放火墙，挡弹幕+灼烧近身）
    V+D → 浴火重生（消失1秒，以强化形态归来）

  雷灵貂：
    V+W → 雷光突袭（瞬移到最近敌人，雷伤+麻痹）
    V+S → 雷盾（你获得雷盾，被攻击时反弹雷伤）
    V+D → 万雷齐发（以你为中心全范围雷电）

  水灵蛟：
    V+W → 潮汐冲击（前方扇形水波，击退+减速）
    V+S → 水灵庇护（你获得3秒30%减伤+持续回血）
    V+D → 沧海之力（全场水波，敌人全减速30%持续3秒）

  木灵蝶：
    V+W → 毒粉弹幕（前方锥形毒粉，叠3层中毒）
    V+S → 生命花环（你获得花环，每秒回2%生命持续5秒）
    V+D → 万木归宗（召唤3株临时灵木环绕你战斗10秒）

  土灵龟：
    V+W → 巨石冲锋（冲向目标，大量击退+石化概率）
    V+S → 龟壳庇护（灵龟变大，你躲后面3秒免伤）
    V+D → 地裂（灵龟砸地，全场减速+石化概率）

  天机灵狐：
    V+W → 幻影突袭（灵狐冲向目标，造成无视防御伤害+标记）
    V+S → 预知之盾（显示接下来3秒所有敌人攻击轨迹）
    V+D → 天机幻境（制造2个你的幻影持续8秒，吸引仇恨）

特殊协同（场景触发）：
  · 你血量<20%：V+S 冷却减半，效果更强
  · Boss暴露弱点：V+W 伤害翻倍
  · 天象匹配灵宠属性：V+D 借天象之力（效果+50%）

灵宠辅助探索：

## 16. 因果系统

### 16.1 因果标记

```
六种标记：善缘/恶行/贪念/道心/执念/逆天
同类累积3个触发因果事件
总标记≥3触发心魔试炼（稀有事件R04）

因果事件：
  善缘×3 → "福报"免费仙品
  恶行×3 → "心魔"精英战+强力Buff
  贪念×3 → "天谴"损30%血换天品
  道心×3 → "顿悟"功法领悟+2层
  执念×3 → "执迷"常用×2其他-20%
  逆天×3 → "天劫提前"额外Boss+境界跳级
```

### 16.2 因果可见性设计

```
设计理念：
  "透明的掌控感"要求玩家知道自己在积累什么因果，但因果
  事件的具体结果仍保持探索性（玩家知道积累了什么，不知道
  触发后会发生什么）。

HUD 显示：
  · 屏幕左下角（生命/灵力条下方）显示当前因果标记图标
  · 每种标记以对应的小图标显示，附带数量
  · 示例：💛×2（善缘2个）  🖤×1（恶行1个）

悬停提示：
  · 暂停时鼠标悬停因果图标，显示：
    "善缘：2/3 — 再行善一次，因果将显化"
  · 不透露具体因果事件内容（保持发现感）

因果标记来源提示：
  · 首次获得每种因果标记时弹出提示：
    "你获得了【善缘】因果标记"
    "善举终有回报……但何时回报，天道自有安排"
  · 后续获得同类标记时，图标闪烁1秒

设计原则：
  · 玩家始终知道"我在积累什么" → 可以有意识地选择方向
  · 玩家不知道"触发后会怎样" → 保持探索和惊喜
  · 第一次触发的因果事件结果隐藏 → 第二次起显示已触发过的结果
```

## 19. 无尽模式

### 19.1 开启条件

```
至少飞升1次 → 主界面解锁"深渊征途"
```

### 19.2 结构

```
每层 3-5 房间（比主线紧凑）
每5层小Boss / 每10层大Boss / 每25层域外天魔 / 每50层天道化身

层数区域：
  1-10 浅层 / 11-20 中层 / 21-30 深层 / 31-40 极层
  41-50 核心 / 51-100 混沌 / 100+ 无尽虚空
```

### 19.3 数值缩放与防溢出

```
基础缩放：
  敌人生命 = base × 1.15^层
  攻击 = base × 1.10^层
  防御 = base × 1.05^层

层级参考：
  第10层生命~404 / 第25层~3292 / 第50层~108366

防溢出四层防线：

  第一层：道韵衰减（见7.2章）
    · 乘算递减收益，防止词条叠加指数爆炸
    · 跨桶构筑收益 > 纯堆单一桶

  第二层：Boss 阶段机制（见18章）
    · 50%血量时无敌切换阶段
    · 天劫之灵独立血条分段
    · 纯DPS无法跳过机制

  第三层：防御公式自然缩放
    · 防御/(防御+100)：防御100→减伤50%，防御200→减伤67%
    · 高层敌人防御增长后，有效血量增长远超面板血量
    · 无尽模式防御=base×1.05^层

  第四层：机制防御系统（31层以上生效）
    · 高层敌人获得"灵甲"：每10层+1%全属性减伤（上限20%）
    · 高层敌人获得"回灵"：每秒回复0.5%最大生命（31+层）
    · 41+层Boss增加1个战斗阶段，51+层增加2个阶段

  第五层：对抗性词条（41层以上生效）
    · 41+层精英/Boss随机获得"劫数词条"
    · 劫数词条示例：
      「铜皮铁骨」减伤15% / 「风行者」移速+30%
      / 「吸血」攻击回血5% / 「反震」受击反弹10%伤害
    · 51+层精英2个劫数词缀，Boss 3个
    · 进入房间时显示劫数词条，玩家可提前规划应对

  第六层：动态难度微调（见23章）
    · 构筑极强时，Boss生命微调+5%

设计意图：
  · 不设伤害上限（构筑成就感不应被数值限制）
  · 通过机制层防御和阶段设计，让高层挑战来自"策略"而非纯数值
  · 对抗性词条增加每局变量，防止Build固化
  · 确保100层以上的战斗仍有趣味性和挑战性
```

### 19.6 每日挑战

```
设计理念：
  作为赛季排行榜的日常补充，给核心玩家每天的理由打开游戏。
  使用"固定种子"保证公平性，限时排名增加紧迫感。

每日挑战规则：
  · 每天UTC 00:00刷新1个新挑战
  · 固定种子：所有玩家当天获得相同的词条/事件/天象序列

## 23. 动态难度系统

### 23.1 设计理念

```
三层难度体系：

  第一层：道心选择（开局，玩家主动选）
  第二层：心魔试炼（游戏内，事件型可选挑战）
  第三层：隐性调节（后台微调，玩家不感知）

三层叠加，让难度"刚好在你能力边缘"。
```

### 23.2 隐性调节（后台）

```
实时评估"构筑完成度"（词条/品质/功法/联动/属性加权）

  构筑很强(>80%)：敌人+5%生命+3%攻击，精英+10%，Boss多1技能
  构筑偏弱(<40%)：敌人-5%生命-3%攻击，商店更友好，Boss血-5%

关键规则：
  · 调节幅度极小（±5%），玩家不感知
  · 只调敌人生命，不调掉落品质（完全公平）
  · 不影响成就和解锁
  · 问道难度调节范围更大（±10%）
  · 证道难度调节范围更小（±3%）
```

│  5.对抗词条│  无尽41+层敌人获得劫数词条（19.3章）              │
│          │  · 敌人也有被动强化（减伤/回血/反弹等）            │
│          │  · 进入房间时可见，玩家可提前规划                  │
├──────────┼──────────────────────────────────────────────────┤
│  6.动态调节│  构筑极强时Boss生命+5%（23.2章）                  │
│          │  · 幅度极小，玩家不感知                            │
│          │  · 最后一道保险                                    │
└──────────┴──────────────────────────────────────────────────┘

设计意图：
  · 每一层都是"软"的——不会让玩家感到被限制
  · 六层叠加后，任何单一Build都无法碾压全部内容
  · 但每一层单独来看，对正常游玩几乎无感知
  · 极端优化的Build仍比普通Build强很多（成就感保留）
  · 100层以上的挑战来自策略和机制应对，而非纯数值对抗
```

## 28. 数据架构

### 28.1 三层架构

```
配表层（策划用）：CSV/Excel，每行一条效果，最简单
  id        name    type   element  quality  trigger    effect      value
  fire_ball 火焰弹  spell  fire     common   on_hit     apply_status burn
  fire_ball 火焰弹  spell  fire     common   on_attack  add_damage   15

枚举层（程序用）：
  enum TriggerType { OnAttack, OnHit, OnKill, OnDodge, Passive, ... }
  enum EffectType { AddDamage, ApplyStatus, ModifyStat, AoeDamage, ... }
  enum Element { Fire, Water, Thunder, Wood, Earth, None, Mixed }
  enum Quality { Common, Rare, Epic, Legendary, Dao }
  enum StatusType { Burn, Freeze, Paralyze, Poison, Slow, ... }

运行时层（引擎用）：
  struct CompiledTag { id, trigger, effect, element, value, ... }
  配表时编译一次，运行时直接用结构体
```
