```
### 1.1 一句话描述

> 一款修仙题材的混合型 Roguelite 游戏，玩家扮演轮回转世的修士，每一局经历从炼气到飞升的完整修仙人生。通过功法搭配、天象利用、地形交互、因果抉择、灵宠并肩、器灵觉醒，打造独一无二的修仙 Build，最终渡劫飞升——或在无尽深渊中挑战极限——或在上古秘境中与其他修士争夺天道机缘，在天象崩塌前带着收获安全撤离。

### 1.2 核心体验

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  动作层 —— 操控施法、闪避腾挪、法术连招（手感爽）        │
│  构筑层 —— 功法搭配、词条联动、Build 成型（策略爽）      │
│  表现层 —— 剑光满屏、雷劫降世、万法归一（视觉爽）        │
│  环境层 —— 天象变化、地形交互、借势而行（策略+操作爽）    │
│  伙伴层 —— 灵宠协同、器灵觉醒、心意相通（陪伴感）        │
│  博弈层 —— 搜索抉择、撤退时机、遭遇应变（紧张爽）        │
│                                                         │

│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 核心卖点（差异化标签）

```
宣传核心标签：

  "天地为棋，万法归一"

  · 天象不是背景，是你的武器
    → 积水导雷、干燥引爆、冰面滑行、毒雾融合
    → 每一局的天象都在改变你的战斗方式

  · 词条不是堆叠，是化学反应
    → 灼烧 × 引燃 × 爆燃 × 火借风势 = 一发子弹烧全场
    → 50+ 种联动组合，玩家自己发现

  · 灵宠不是摆设，是战友
    → V+方向键触发协同技能
    → 火灵雀冲锋留火带、雷灵貂瞬移雷击、土灵龟为你挡刀

  · 死亡不是结束，是下一世的开始
    → 前世遗泽引导方向、轮回点数永久成长、因果系统延续

  · 每局都是不同的修仙人生
    → 天命种子暗中影响机缘走向、结构化随机生成确保每局不重复
    → 同样的灵根，走出完全不同的道路

  · 道统不是彩蛋，是修炼的里程碑
    → 凑齐特定词条组合，觉醒"道统"——称号、特效、强化三位一体
    → 每个角色有专属道统，解锁终极飞升结局

  · 天道机缘不止你一人在争
    → 2-3 支修士队伍同时进入上古秘境
    → 可以合作、可以夺缘、可以各走各路
    → 天象崩塌前，带着收获安全撤离——或在撤离点伏击对手
```

### 1.4 目标受众

- **主要：** 肉鸽核心玩家（Hades / 杀戮尖塔 / 以撒的结合）
- **次要：** 修仙/仙侠文化爱好者
- **潜在：** 喜欢 Build 构筑和数值爆炸的休闲玩家
- **潜在：** 搜打撤/Extraction 类玩家（逃离塔科夫 / Dark and Darker 爱好者）
- **长期：** 无尽模式排行榜硬核玩家

### 1.5 竞品分析

| 竞品 | 学习的 | 不同的 |
|------|--------|--------|
| Hades | 打击感、叙事融合、斩杀特写 | 天象地形交互、因果系统、道统构筑 |
| 杀戮尖塔 | 词条联动深度、结构化随机地图 | 动作而非卡牌，灵宠/器灵 |
| 以撒的结合 | 道具池组合爆炸、房间随机生成 | 修仙主题、功法领悟 |
| 暖雪 | 修仙美术风格 | 天象系统、灵宠协同、无尽模式 |
| 吸血鬼幸存者 | 弹幕密度、数值膨胀 | 主动操作和策略深度 |
| 修仙模拟器 | 天命/出身随机开局 | Roguelite 节奏而非模拟经营 |
| Dark and Darker | 搜打撤核心循环、队伍协作、撤离决策 | 修仙题材、天象恶化时间压力、因果系统 |
| 逃离塔科夫 | 局外准备+局内搜索+撤离博弈、PvPvE | 俯视角动作而非FPS、灵宠探路替代侦察 |

## 2. 核心理念与设计哲学

### 2.1 四个心理钩子

```
钩子 1：可能性的诱惑
  "这局说不定能抽到那个逆天功法……"
  → 随机奖励、隐藏内容、小概率事件、灵宠/器灵

钩子 2：差一点的遗憾
  "就差一个词条就凑成完美 combo 了！"
  → Combo 指示器让玩家看到"完成度 75%"的 build

钩子 3：下次会更好的掌控感
  "我知道该怎么选了，再来一局肯定更强"
  → 前世遗泽 + 局外成长 + 系统理解加深

钩子 4：无尽的攀登
  "我还能更强……深渊第 50 层见"
  → 无尽模式的排行榜和极限挑战

钩子 5：贪婪与恐惧的交织
  "再搜一个区域就撤……但万一有好东西呢？"
  → 上古秘境中搜打撤的核心决策张力
```

### 2.2 七项设计原则

```
1. 选择大于运气
   → 所有随机都给玩家"选择权"
   → 难度让玩家自己选（开局选择 + 心魔试炼）
   → 路径选择始终提供2-3个有意义的选项

2. 发现大于告知
   → 词条联动不直接告知，让玩家自己发现 combo
   → 天象 × 地形 × 词条的交互等待玩家探索
   → 首次天象交互通过场景演出引导，而非弹窗教程
   → 隐藏房需要观察或灵宠探路发现

3. 节奏大于强度
   → 不能一直爽，要有弱→强→碾压的节奏
   → Boss 门前有"呼吸点"，战斗内有高潮低潮
   → 天机一击特写不打断节奏（0.3-1秒慢动作内完成）
   → 关卡骨架保证每关有清晰的难度弧线

4. 死亡也是收获
   → 前世遗泽引导下一局方向
   → 轮回点数永久成长
   → 名场面回顾让死亡有记忆点

5. 修仙即肉鸽
   → 轮回/境界/功法/渡劫/飞升 全部对应肉鸽机制

6. 每一秒都有意义
   → 战斗结束直接选路径（无跑图）
   → 过渡动画同时传递信息
   → Boss 门前暂停是"有意义的调息"
   → 秘境中每一步移动都有搜索价值或暴露风险

7. 透明的掌控感
   → 难度玩家自己选（开局 + 心魔试炼）
   → Build 可视化五层体系（含道统里程碑）
   → 灵宠/器灵成长全程可见
   → 因果标记 HUD 可见（但触发结果仍需探索）
   → 道韵衰减机制可在伤害面板查看当前衰减状态
```

## 3. 难度系统

### 3.1 设计理念

```
难度系统的核心矛盾：
  · 新手需要低门槛进入 → 不能一上来就难
  · 老手需要挑战感     → 不能一直简单
  · 玩家讨厌"被看低"   → 不能用"简单/普通/困难"这种标签
  · 玩家喜欢掌控感     → 让玩家自己选，而非系统暗改

解决方案：用修仙叙事包装难度选择

  不是"选难度"
  而是"选择你的道心"
```

### 3.4 难度选择的心理设计

```
关键原则：不要让玩家觉得"选简单 = 丢人"

  1. 没有"简单/普通/困难"这种直白标签
     → 用修仙术语包装："问道/悟道/证道"
     → 每个都有正面描述，没有贬义

  2. 每个难度都有独特价值
     → 问道：适合体验剧情和学习系统
     → 悟道：标准修仙体验
     → 证道：有额外奖励和称号
     → 不是"弱者选简单"，而是"不同道心不同路"

  3. 可以随时修改
     → 设置里随时切换（下一层生效）
     → 不会被"锁定"在某个难度
     → 鼓励玩家"先问道入门，再证道挑战"

  4. 无尽模式难度独立
     → 无尽模式不使用道心系统
     → 而是用层数自然递增难度
     → 硬核玩家的挑战在无尽模式中

```
### 7.2 伤害流水线（9阶段 + 道韵衰减）

```
完整伤害计算流程（PvE）：

  1.基础伤害 → 2.属性加成 → 3.词条加算
  → 4.词条乘算（经道韵衰减修正）→ 5.环境乘算（经道韵衰减修正）
  → 6.灵宠/器灵加成（经道韵衰减修正）→ 7.暴击
  → 8.防御减免 → 9.修正
  → 10.后处理（触发效果+天机一击判定+显示数字）

PvP伤害流水线（秘境模式专用，见第20章）：
  基础伤害 → 属性加成 → 词条加算 → 词条乘算(×0.6) → 环境乘算(×0.8)
  → 灵宠/器灵加成(×0.7) → 暴击 → PvP韧性减免 → 伤害上限裁切 → 最终伤害

道韵衰减机制：

  设计意图：
    防止多个乘算来源线性叠加导致伤害指数爆炸，同时
    保留"每多凑一个词条都有收益"的构筑满足感。
    采用递减收益（而非硬上限），玩家仍可自由追求极限。

  伤害桶分类（4个桶）：
    桶A·词条乘算：联动词条效果（纯阳真火、焚天、五行归元等）
    桶B·环境乘算：天象加成、地形加成
    桶C·伙伴乘算：灵宠被动加成、器灵加成
    桶D·境界乘算：境界天赋、局外成长加成

  同桶内叠加规则：
    第1个乘算来源：100% 效果
    第2个乘算来源：80% 效果（即超出基准的部分×0.8）
    第3个及以上：  60% 效果（即超出基准的部分×0.6）

  计算示例：
    桶A有2个乘算来源：纯阳真火(×1.3) + 焚天(×1.5)
      纯阳真火：×1.3（第1个，100%效果）
      焚天：1 + (1.5-1) × 0.8 = ×1.4（第2个，80%效果）
      桶A总乘算：×1.3 × 1.4 = ×1.82

    vs 无递减：×1.3 × 1.5 = ×1.95
    差异7%，感知不明显

    如果桶A有第3个乘算(×1.3)：
      有递减：1 + (1.3-1) × 0.6 = ×1.18 → 桶A总计 ×1.3 × 1.4 × 1.18 = ×2.15
      无递减：×1.3 × 1.5 × 1.3 = ×2.54
      差异15%，有效防止溢出

    最终伤害 = 基础伤害 × 桶A × 桶B × 桶C × 桶D × 暴击

  设计效果：
    · 桶间相乘（鼓励跨系统构筑：词条+天象+灵宠+境界）
    · 同桶递减（防止纯堆单一来源的极端数值）
    · 3个桶各叠2个乘算 > 1个桶叠6个乘算
    · 自然引导玩家关注天象利用和灵宠搭配（跨桶收益更高）
    · 每多拿一个词条仍有收益（不设硬上限），但边际递减
```

## 8. 天象与地形交互系统

### 8.1 设计理念

```
天象不是数值 buff，而是改变战斗环境的物理力量。

核心原则：
  · 天象创造可交互的地形元素
  · 每种地形只有 1-2 个核心交互规则
  · 新手可以忽略（不强制利用），高手利用地形打出极限
  · 视觉提示极其清晰，不需要看文字说明
  · 逐步引入：第1关只有数值效果，第2关开始地形交互
  · 首次交互通过场景演出引导，不依赖弹窗
  · 上古秘境中天象随时间恶化（非房间推进），见第20章
```

### 8.4 天象变化节奏

```
主线模式：
  第1关：1种天象，只有数值效果（学习期）
  第2关：每3-4层变化，地形交互开始（适应期）
  第3关：每2-3层变化（熟练期）
  第4关：每1-2层变化（挑战期）
  第5关：每层不同（极限期）

无尽模式：
  每3层变化，51+层出现混沌天象

上古秘境模式：
  天象随时间恶化（非房间推进），详见第20章20a.6节
  灵雨(0-5min) → 烈日(5-10min) → 雷暴(10-15min) → 蚀月(15-20min) → 混沌崩塌(20min+)

天象预告：
  · 问道难度：提前 2 层预告
  · 悟道难度：提前 1 层预告
  · 证道难度：当层才知道（进入新层时，天象交互速查卡自动
    弹出3秒，帮助玩家快速识别当前天象的最佳应对策略）
  · 上古秘境：天象恶化进度条始终可见，当前天象+下一阶段天象可见
  · 局外升级可额外提前 1 层（主线/无尽）
```

## 12. 灵宠系统

### 12.1 灵宠概览

```

```
### 12.2 灵宠获取

```

  天象/因果专属事件 / 无尽每20层奖励

```
### 12.3 灵宠获取保底机制

```
设计理念：
  灵宠是核心体验伙伴层的支柱（见1.2），不应出现连续多局
  无法获取灵宠的情况。保底机制隐性运行，玩家不直接感知。

保底规则：
  · 前3关（炼气→金丹）中，至少有 1 次灵宠获取机会触发
  · 若到第3关Boss战结束仍未获得任何灵宠，则Boss战后
    必定出现"灵兽巢穴"事件（从5种基础灵宠中选择1只）
  · 保底不告知玩家（保持"发现大于告知"原则）

前世遗泽联动：
  · 若上一局拥有灵宠但角色死亡，前世遗泽中增加选项：
    "灵宠魂魄" → 开局获得随机基础灵宠蛋（见21.2章）
```

### 12.5 灵宠协同系统（核心交互）

```
设计理念：
  · 你始终控制你的角色（不切换控制）
  · 灵宠自动战斗（现有系统）
  · V+方向键触发"协同动作"（15-30秒冷却）
  · 每种灵宠的协同动作不同
  · 操作简单，不打断战斗节奏

操作方式：
  V + W（前）：灵宠冲锋 — 冲向鼠标方向，造成伤害+击退
  V + S（后）：灵宠护卫 — 环绕你，5秒内为你挡弹幕
  V + A（左）：灵宠探路 — 灵宠前冲，照亮迷雾/提示隐藏
  V + D（右）：灵宠爆发 — 灵宠释放强化版技能，之后10秒虚弱

每种灵宠的具体协同动作：

  火灵雀：
    V+W → 火焰冲锋（冲向目标，路径留火焰带3秒）
    V+S → 烈焰守护（环绕你释放火墙，挡弹幕+灼烧近身）
    V+D → 浴火重生（消失1秒，以强化形态归来）

  雷灵貂：
    V+W → 雷光突袭（瞬移到最近敌人，雷伤+麻痹）
    V+S → 雷盾（你获得雷盾，被攻击时反弹雷伤）
    V+D → 万雷齐发（以你为中心全范围雷电）

  水灵蛟：
    V+W → 潮汐冲击（前方扇形水波，击退+减速）
    V+S → 水灵庇护（你获得3秒30%减伤+持续回血）
    V+D → 沧海之力（全场水波，敌人全减速30%持续3秒）

  木灵蝶：
    V+W → 毒粉弹幕（前方锥形毒粉，叠3层中毒）
    V+S → 生命花环（你获得花环，每秒回2%生命持续5秒）
    V+D → 万木归宗（召唤3株临时灵木环绕你战斗10秒）

  土灵龟：
    V+W → 巨石冲锋（冲向目标，大量击退+石化概率）
    V+S → 龟壳庇护（灵龟变大，你躲后面3秒免伤）
    V+D → 地裂（灵龟砸地，全场减速+石化概率）

  天机灵狐：
    V+W → 幻影突袭（灵狐冲向目标，造成无视防御伤害+标记）
    V+S → 预知之盾（显示接下来3秒所有敌人攻击轨迹）
    V+D → 天机幻境（制造2个你的幻影持续8秒，吸引仇恨）

特殊协同（场景触发）：
  · 你血量<20%：V+S 冷却减半，效果更强
  · Boss暴露弱点：V+W 伤害翻倍
  · 天象匹配灵宠属性：V+D 借天象之力（效果+50%）

灵宠辅助探索（主线+秘境通用）：
  · 使用 V+A 探路时，灵宠冲向方向有隐藏墙壁会"叫"一下
  · 秘境中V+A额外功能：检测敌方玩家位置（发出"警觉"声）
    / 检测守卫巡逻方向（发出"低吼"声）
    / 发现隐藏宝箱（发出"兴奋"声）
  · 三种声音不同，玩家需要学会辨别
```

## 13. 器灵系统

### 13.1 器灵概览

```

  · 特定条件"共鸣"觉醒
  · 觉醒后法宝能力质变：被动→主动技能+人格
  · 有性格和战斗建议（可选关闭）
  · 可成长
  · 一局最多1个器灵觉醒
```

### 13.2 觉醒条件

```
五种共鸣：
  1. 战斗共鸣：使用法宝效果累计N次
  2. 天象共鸣：特定天象战斗N层
  3. 属性共鸣：某属性达阈值
  4. 事件共鸣：特殊事件触发
  5. 因果共鸣：因果标记达条件

觉醒：满足条件→法宝发光→提示→按F注入灵力→3秒动画

器灵觉醒保底机制：
  · 前4关（炼气→元婴）中，至少有 1 次觉醒条件可被触发
    的机会出现
  · 若到第4关结束仍未觉醒任何器灵，第4关Boss前的调息
    阶段必定出现"觉醒指引"事件（消耗100灵石，对已装备
    法宝强制进入共鸣状态，觉醒进度直接提升至80%）
  · 保底不告知玩家，但确保每局至少有1次器灵觉醒机会
```

### 14.3 关卡生成系统（结构化随机）

```
设计理念：
  采用"结构化随机"方案——骨架固定保证节奏感，内容随机保证
  每局不重复。不使用完全手绘（重玩价值低）或完全随机
  （节奏失控），而是用规则约束随机。

一局游戏的完整关卡生成流程：

  玩家选择道心+灵根+角色+天命种子
      ↓
  系统生成"本局种子"（所有随机的根源，可用于分享和复现）
      ↓
  对每一关（第1关→第5关）预生成全部房间：
    ┌─────────────────────────────────────────────────────────┐
    │  步骤1：确定骨架                                         │
    │    · 房间总数（按关卡配置：6-8个房间槽位）              │
    │    · 最后1个槽位固定为Boss前调息                        │
    │    · 第1个槽位固定为普通战斗房（引入新关卡主题）        │
    │    · 标记必出槽位：每关至少1个精英房+至少1个事件/坊市   │
    │                                                         │
    │  步骤2：按骨架规则，逐槽抽取房间类型                     │
    │  步骤3：检查防连续规则，必要时重随                       │
    │  步骤4：确定每层天象                                     │
    │  步骤5：为每个战斗房间生成内部布局                       │
    │  步骤6：为每个战斗房间确定敌人组合+波次                  │
    │  步骤7：为每个事件房抽取事件                             │
    │  步骤8：为每个坊市抽取商品                               │
    │  步骤9：标记隐藏房                                       │
    └─────────────────────────────────────────────────────────┘
      ↓
  全部5关预生成完毕
      ↓
  玩家开始第1关 → 逐房间推进

为什么全预生成而非逐房间生成？
  · 保证天象预告的准确性
  · 保证隐藏房位置的一致性
  · 保证关卡整体节奏的连贯性
  · 保证保底机制可执行
  · 性能无压力：预生成5关×8房≈40个房间，数据量极小
```

### 14.5 房间内布局生成

```
设计理念：
  每个战斗房间的内部布局也是随机的，但有严格约束确保
  "可玩性"优先于"随机性"。

固定部分：
  · 房间尺寸（按关卡递增）：
    第1关 600×400 → 第2关 700×450 → 第3关 800×500
    → 第4关 850×550 → 第5关 900×600

随机部分：
  · 障碍物（2-4组，按关卡主题）
  · 地形元素（受天象影响）
  · 敌人出生点（5种布局模式随机）

随机约束规则（保证可玩性）：
  · 障碍物不阻塞主要移动路径（至少保留2条宽度>60像素的通道）

  · 大型障碍物（>80像素）不超过2个/房间

```
### 16.1 因果标记

```

秘境中因果事件即时触发（累积3个同标记时立即生效），效果更强。
```

### 16.2 因果可见性设计

```
设计理念：
  "透明的掌控感"要求玩家知道自己在积累什么因果，但因果
  事件的具体结果仍保持探索性。

HUD 显示：
  · 屏幕左下角（生命/灵力条下方）显示当前因果标记图标
  · 每种标记以对应的小图标显示，附带数量
  · 示例：💛×2（善缘2个）  🖤×1（恶行1个）

悬停提示：
  · 暂停时鼠标悬停因果图标，显示：
    "善缘：2/3 — 再行善一次，因果将显化"
  · 不透露具体因果事件内容（保持发现感）

设计原则：
  · 玩家始终知道"我在积累什么"
  · 玩家不知道"触发后会怎样"
  · 第一次触发的因果事件结果隐藏 → 第二次起显示已触发过的结果

中文标题：上古秘境
副标题：机缘有限，道友亦敌亦友

一句话描述：
  2-3支修士小队（每支1-4人）同时进入同一处上古秘境，
  搜索机缘、击败守卫、对抗其他队伍——
  在天象崩塌之前，带着收获安全撤离。
  你可以在入口处与道友结伴，也可以在秘境深处背刺同伴。

```
### 20.0.2 设计理念

```
搜打撤核心：

  搜索（搜）—— 秘境中散落着宝箱、灵矿、遗迹、隐藏洞穴
    → 越深入越危险，但奖励越丰厚
    → 灵宠探路是搜索的核心工具
    → 时间压力来自天象恶化，不是倒计时器

  战斗（打）—— 秘境守卫（PvE）+ 其他修士队伍（PvP）
    → PvE是日常，PvP是变量
    → 可以主动避战——绕开巡逻守卫、避开其他队伍
    → 但Boss和宝藏区域是必争之地

  撤离（撤）—— 在天象崩塌前带出搜刮物
    → 撤离点有限且需要引导时间
    → "再搜一个区域还是现在就撤"——永恒的决策
    → 最后30秒撤离点会暴露在所有队伍视野中

PvP设计理念：

  · PvP不是目的，是手段——你杀人是为了抢夺对方搜刮的资源
  · PvP有成本——战斗产生"灵力波动"，会吸引秘境守卫和天象加速恶化
  · PvP有风险——你可能打不过，也可能被第三方渔翁得利
  · PvP有选择——你可以全程不打玩家，纯PvE搜索撤离
  · 但最肥的资源点（Boss房/终极宝箱）往往有多支队伍争抢

  修仙叙事包装：
  · 不是"击杀其他玩家"，是"夺缘"——争夺有限的天道机缘
  · 不是"敌方玩家"，是"争缘的道友"——亦敌亦友
  · 秘境崩塌时的"最后争夺"是"天道不容，各凭本事逃出生天"
```

### 20.0.3 核心循环

```
  准备阶段
    · 选择灵根+角色+灵宠
    · 从局外仓库选择带入装备（法宝+词条+丹药，数量受道心限制）
    · 天命种子揭示
      ↓
  进入秘境
    · 2-3支队伍同时传送进入（互不可见出生点）
    · 秘境地图生成（随机，15-25个区域节点）
    · 天象为初始状态（灵雨，有利）
      ↓
  ┌─→ 搜索阶段（搜）────────────────────────────────────────┐
  │  · 在区域节点间移动，探索未知区域                          │
  │  · 搜索宝箱、击败守卫、触发事件                           │
  │  · 灵宠协助探路（发现隐藏/提示危险/检测敌人）             │
  │  · 天象随时间逐步恶化（5分钟/级）                         │
  │  · 其他队伍也在搜索——可能在同一区域遭遇                    │
  │     ↓                                                      │
  ├─→ 战斗阶段（打）────────────────────────────────────────┤
  │  · PvE：秘境守卫（巡逻型精英/区域Boss）                   │
  │  · PvP：与其他队伍遭遇——战斗/谈判/回避                    │
  │  · PvP战斗产生"灵力波动"→吸引守卫+加速天象恶化           │
  │  · 击败对方→"夺缘"→获得对方50%搜刮物                    │
  │     ↓                                                      │

```
#### 20.1.3 地图生成流程

```
  步骤1：确定拓扑骨架
    · 生成15-25个节点的有向图（无环，从边缘到中心有路径趋势）
    · 每个节点至少有2条连接
    · 关键约束：任何两个出生点到Boss区域的距离差<2个节点

  步骤2：分配区域类型
    · Boss区域固定在中心或最深处
    · 撤离区域固定在边缘（3个，分散放置）
    · 宝藏区域2-3个，分布在Boss区域附近
    · 灵泉区域2-3个，分布在中段
    · 其余从搜索/战斗/事件中随机分配

  步骤3：为每个区域生成内部布局
    · 复用主线房间布局系统，尺寸统一为大房间（900×600）

  步骤4：确定守卫组合
    · 搜索区域：1-2个普通守卫
    · 战斗区域：1个精英+2-3个杂兵
    · 宝藏区域：2个精英+3-4个杂兵
    · Boss区域：秘境Boss

  步骤5：确定出生点
    · 2-3支队伍，出生在地图边缘不同方向
    · 出生区域自动设为灵泉

  步骤6：迷雾覆盖
    · 所有区域初始为迷雾状态
    · 与已探索区域相邻的1层区域可见（类型可见但内容未知）
    · 灵宠探路可额外揭开1层迷雾
```

### 20.3 战斗系统（PvE + PvP）

#### 20.3.1 PvE——秘境守卫

```

    · 血量极高（相当于主线Boss4的200%）

  设计意图：
  · 普通守卫是"可以打可以绕"的选择题
  · 精英守卫是"需要认真对待"的战术挑战

```
#### 20.3.3 PvP伤害模型

```
  PvP伤害独立于PvE，需单独平衡。

```
### 20.4 撤离系统

#### 20.4.1 撤离点机制

```
  撤离点数量：3个，分布在地图边缘不同位置

  撤离点状态周期（每3分钟循环）：
    · 封闭期（2分钟）—— 不可使用，显示冷却倒计时
    · 开放期（1分钟）—— 可引导撤离

  撤离引导：
    · 到达撤离点后按G开始引导（15秒）
    · 引导期间无法移动和攻击（队友可保护）
    · 引导完成时，引导者+100像素内所有队友一起撤离
    · 引导期间被打断（受到伤害/被控制）→ 中断，需重新开始

  撤离点冲突：
    · 撤离点附近可以PvP（不是安全区）
    · 多支队伍可能同时想用同一个撤离点
    · 最后30秒（天象崩塌前）所有撤离点强制开放+全地图高亮
    → 这是PvP最激烈的时刻——所有队伍涌向撤离点
```

#### 20.4.2 天象崩塌与强制撤离

```
  秘境有总时间限制，由天象恶化进度决定：

  ┌──────────────────────────────────────────────────┐
  │  天象恶化进度条（屏幕右上角，所有玩家可见）      │
  │                                                   │
  │  ████████████████░░░░░░  72%  蚀月阶段           │
  │                                                   │
  │  当前天象：蚀月                                   │
  │  下一阶段：混沌崩塌                               │
  │  预计崩塌：4:23                                   │
  └──────────────────────────────────────────────────┘

  崩塌倒计时最后60秒：
    · 屏幕边缘出现裂纹特效
    · 所有撤离点强制开放+全地图高亮
    · 混沌天象叠加生效（多重天象+逆天象）
    · 所有守卫暴走（属性+50%，无差别攻击）
    · 大字："秘境将崩，速速撤离！"

  崩塌倒计时最后15秒：
    · 全屏红色警告
    · 所有未撤离玩家持续掉血（每秒5%最大生命）
    · 撤离引导时间减半（7.5秒）

  崩塌时刻（0秒）：
    · 所有未撤离玩家强制道消
    · 丢失全部搜刮物，无救援机会
```

#### 20.4.3 撤离成功与失败

```
  撤离成功：
  · 保留全部搜刮物
  · 结算时转化：
    · 灵石 → 轮回点数（1:0.5）
    · 词条 → 词条图鉴解锁进度
    · 法宝碎片 → 集齐3个合成完整法宝
    · 丹药 → 局外丹药储备
    · 道品词条碎片 → 集齐3个合成道品词条

  出局（道消或崩塌未撤离）：
  · 丢失60%搜刮物（随机选择丢失哪些）
  · 带入的装备全部消耗
  · 仍获得轮回点数（基础保底+已击杀守卫奖励）
  · 获得"名场面回顾"

  存活但队友出局：
  · 你撤离成功，队友出局
  · 可选择将部分搜刮物"托付"给出局队友（他们获得30%）

```
#### 20.5.2 天象对PvP的影响

```
  灵雨阶段（0-5分钟）：
  · 积水区域存在 → 雷系可利用积水导电控制区域
  · PvP价值：低（双方都在搜索，冲突少）

  烈日阶段（5-10分钟）：
  · 干燥区域出现 → 火系爆燃成为区域封锁手段
  · PvP价值：中（争抢资源点）

  雷暴阶段（10-15分钟）：
  · 天雷频繁 → 移动更危险，雷系伤害爆表
  · PvP价值：高（资源稀缺，冲突频繁）

  蚀月阶段（15-20分钟）：
  · 全伤+20% → PvP更激烈
  · CD-30% → 技能更频繁
  · PvP价值：极高（撤离点争夺）

  设计意图：
  · 天象为PvP提供"环境变量"——不同天象下最优PvP策略不同
  · 灵根多样性在多人模式下价值大增
  · "等天象变了再打"成为一种策略
```

## 21. 失败与重生系统

### 21.1 设计理念

```
死亡不是惩罚，而是下一次轮回的起点。
三层缓冲让玩家"死了也想再来"：
  1. 前世遗泽（下局方向引导）
  2. 轮回点数（永久成长资源）
  3. 名场面回顾（死亡也有高光记忆）

上古秘境中死亡（出局）的缓冲层：
  1. 搜刮物保留40%（问道70%/悟道40%/证道20%）
  2. 击杀守卫的轮回点数保底
  3. 名场面回顾

```
## 24. 动态难度系统

### 24.1 设计理念

```
三层难度体系：

  第一层：道心选择（开局，玩家主动选）
  第二层：心魔试炼（游戏内，事件型可选挑战）
  第三层：隐性调节（后台微调，玩家不感知）
```

### 24.2 隐性调节（后台）

```
实时评估"构筑完成度"（词条/品质/功法/联动/属性加权）

  构筑很强(>80%)：敌人+5%生命+3%攻击，精英+10%，Boss多1技能
  构筑偏弱(<40%)：敌人-5%生命-3%攻击，商店更友好，Boss血-5%

关键规则：
  · 调节幅度极小（±5%），玩家不感知
  · 只调敌人生命，不调掉落品质（完全公平）
  · 不影响成就和解锁
  · 问道难度调节范围更大（±10%）
  · 证道难度调节范围更小（±3%）
  · 上古秘境中动态难度微调秘境守卫强度（不影响PvP）
