# 《轮回仙途》UI/UX 美术设计规范 v1.0

**文档类型：** UI/UX + 界面美术指导  
**适用版本：** Godot 4.6 客户端（`game/`）  
**设计依据：** GDD v6.0 §24–25、现有原型界面、竞品（暖雪 / Hades / Slay the Spire）  
**最后更新：** 2026-06-09

---

## 1. 文档目的

本规范供 **UI 设计师、界面美术、前端/Godot 实现** 统一使用，解决当前原型「功能齐全但视觉占位」的问题，建立：

- 可落地的 **Design Token**（色、字、间距、圆角、层级）
- 全游戏 **界面清单 + 布局线框 + 状态说明**
- **组件库** 与 **动效/反馈** 标准
- 与代码中已有面板的 **对照表与迭代优先级**

> 玩家可见文案继续使用 GDD 术语（真元、灵力、灵石、道消…）；代码层仍用英文标识符。

---

## 2. 设计目标

| 维度 | 目标 | 衡量标准 |
|------|------|----------|
| 可读性 | 弹幕战斗中 0.5 秒内读懂 HP / 冷却 / 天象 | 核心信息不超过 3 个视觉焦点区 |
| 仙侠气质 | 水墨暗色 + 金线点缀，避免「通用 RPG 紫蓝 UI」 | 主色板与 GDD §25 一致 |
| 构筑可读 | 词条 / Combo / 道统 / 法术槽关系一眼可辨 | Build 区与战斗区分离 |
| 发现感 | 天象交互、Combo、法术解锁靠 **演出** 而非弹窗教程 | 首次交互有慢动作 + Toast |
| 节奏 | 选择界面（词条/突破/择路）有「呼吸感」，战斗 HUD 偏紧凑 | 见 §7 密度档位 |

**设计参数（内部拨盘）：**

- **布局变化度 7/10** — 左信息 / 右天象 / 底技能栏，非对称；全屏弹层居中
- **动效强度 6/10** — 习得 Toast、天机一击、Boss 阶段有过场；菜单过渡 200–350ms
- **信息密度 — 战斗 8/10，meta 4/10** — 战斗 HUD 接近「修士仪表盘」；轮回殿/Setup 留白更多

## 3. 视觉识别系统

### 3.1 色彩 Token

与 GDD §25 对齐，所有 UI 禁止硬编码散落色值，统一引用 Token 名。

#### 基础面

| Token | Hex | 用途 |
|-------|-----|------|
| `bg.deep` | `#0D0D0D` | 全屏遮罩、最深背景 |
| `bg.panel` | `#1A1A2E` | 面板、卡片底 |
| `bg.panel_alt` | `#2D2D44` | 次级分区、输入框底 |
| `text.primary` | `#F0ECE4` | 主正文 |
| `text.secondary` | `#C4B69C` | 说明、次要标签 |
| `text.muted` | `#8A8278` | 禁用、占位 |
| `accent.gold` | `#FFD700` | 标题、道统、突破 |
| `accent.gold_soft` | `#F0D68A` | 金色次级强调 |

#### 属性色（战斗 / 词条 / 法术）

| 元素 | Hex | 用法 |
|------|-----|------|
| `elem.fire` | `#FF6B35` | 火系词条、Q 烈焰弹、灼烧 |
| `elem.water` | `#4ECDC4` | 水系、R 玄冰、减速 |
| `elem.thunder` | `#FFD700` | 雷系、E 雷击、麻痹 |
| `elem.wood` | `#7BC67E` | 木系、毒、灵宠辅助 |
| `elem.earth` | `#C4A35A` | 土系、防御、护体 |
| `elem.chaos` | `#B57EDC` | 混沌 / 五行归元（少用） |

#### 品质色（词条边框 + 标签）

| 品质 | 玩家文案 | Hex | 边框/光晕 |
|------|----------|-----|-----------|
| 凡 | common | `#B0B0B0` | 1px 实线 |
| 灵 | rare | `#4E9AF1` | 1px + 弱外发光 |
| 仙 | epic | `#A855F7` | 2px + 粒子点缀 |
| 天 | legendary | `#F59E0B` | 2px 金边 + 扫光 |
| 道 | dao | `#EF4444` | 2px 金红渐变边 |

#### 语义色

| Token | Hex | 用途 |
|-------|-----|------|
| `state.hp` | `#E85D5D` | 真元条、受伤闪红 |
| `state.mana` | `#5B9BD5` | 灵力条 |
| `state.buff` | `#7BC67E` | 机缘 / 正向 |
| `state.debuff` | `#C45C5C` | 劫数 |
| `state.learn` | `#FFD700` | 习得 Toast（spell） |
| `state.rebind` | `#8EC5FF` | 换绑 Toast |
| `state.skill` | `#A6F0C6` | 功法精进 Toast |

**对比度：** 正文与 `bg.panel` 对比度 ≥ **4.5:1**；字号 ≤ 12px 时 ≥ **7:1**。

### 3.2 字体

| 层级 | 中文推荐 | 拉丁/数字 | 字号 (1080p) | 字重 |
|------|----------|-----------|--------------|------|
| Display | 思源宋体 / 方正清刻本悦宋 | — | 26–32px | Bold |
| Title | 思源黑体 CN Bold | — | 20–24px | Bold |
| Body | 思源黑体 CN Regular | Inter / Noto Sans | 14–16px | Regular |
| Caption | 同上 | 同上 | 12–13px | Regular |
| Number | 思源黑体 / JetBrains Mono | Mono | 14–18px | Medium |

**规则：**

- 标题可用 **宋体 + 金描边 1px** 强化仙侠感；正文、数值用黑体保证可读
- 伤害数字、冷却秒数、灵石数量用 **等宽数字** 避免跳动
- 禁止正文中使用 Emoji 代替图标（GDD 线框中的 emoji 仅作占位）

### 3.3 间距与圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `space.xs` | 4px | 图标与文字间距 |
| `space.sm` | 8px | 行内、标签间距 |
| `space.md` | 12px | 卡片内边距 |
| `space.lg` | 16px | 面板内边距（当前 Margin 16） |
| `space.xl` | 24px | 大面板、Setup |
| `radius.sm` | 4px | 进度条、小标签 |
| `radius.md` | 8px | 按钮、卡片 |
| `radius.lg` | 12px | 大弹窗 |

### 3.4 图标

- **风格：** 线性 1.5px，圆角端点，填充仅用于属性/品质小徽章
- **尺寸：** 16 / 20 / 24 / 32px 四档
- **来源建议：** 自制 SVG 或 Phosphor 风格自绘；元素图标与 `elem.*` 色一致
- **禁止：** 混用 3D 图标与扁平线稿

### 3.5 材质与装饰

- **面板：** 深色底 + **内描边 1px `#FFFFFF` @ 8%** + 可选宣纸纹理 `@ 4%` 叠加
- **道统 / 突破：** 金色卷轴角饰（四角 L 形金属线，非全框）
- **分割线：** 1px `#C4B69C` @ 25%，或极细金线渐变
- **避免：** 大面积毛玻璃、霓虹紫渐变、通用「游戏 UI 六边形框」

## 5. 核心界面规范

### 5.1 战斗 HUD（UI-03）

**布局（1920×1080 基准）：**

```
┌──────────────────────────────────────────────────────────────────┐
│ [左上·修士信息区 320px]              [右上·天象区 280px]          │
│  · 关卡/波次 Title (gold)            · 天象名 + 图标              │
│  · 真元条 + 数字                     · 下一天象预告（若有）       │
│  · 连击                               · 因果/试炼摘要（单行）     │
│  · 境界 + 词条槽位                    │                          │
│  · 道路 Build 一行摘要                │                          │
│  · 道统 / Combo 轨道                  │                          │
│  · 灵石                               │                          │
│  · 功法进度                           │                          │
│  · Q/E/R 法术（各一行，未解锁灰显）   │                          │
│                                      [中央·LearnToast 480px]    │
│                                      [中下·伤害飘字]              │
│ [底部·操作提示 单行 muted]                                        │
└──────────────────────────────────────────────────────────────────┘
```

**与当前原型差异：**

| 项目 | 现状 | 目标 |
|------|------|------|
| 信息区 | 左上一长列 VBox 17+ 行 | 分区折叠：战斗核心 / Build 详情（Tab 或展开） |
| 真元条 | 默认 ProgressBar | 自定义：渐变填充 + 受击闪白 + 低血量脉动 |
| 法术栏 | 纯文本三行 | 图标 + 键位徽章 + 冷却环 |
| LearnToast | Label 26px 居中 | 卷轴底图 + 金色描边 + 0.3s 缩放入场 |
| 灵宠 | 单行文字 | 小头像 + 协同 CD 环 |

**HUD 显示开关（暂停菜单，已实现逻辑）：**

- 敌人血条 / 伤害飘字 — 保持现有 `SaveManager` 设置项

### 5.2 词条三选一（UI-04）

**结构：** 全屏 dimmer `@ 55%` → 居中面板 `760×360` → 标题 → 灵石 → **三卡横排**

**卡片组件 `AffixCard`：**

```
┌─────────────────┐
│ [属性图标 32px]  │
│  词条名 Title    │
│  凡品 · 法术     │  ← 品质色标签
│  ─────────────  │
│  描述 2–3 行     │
│  Combo 标签行    │
│  [ 择此机缘 ]    │  ← 主按钮，hover 品质色边
└─────────────────┘
```

- 卡片宽 **220px**，间距 12px，选中 hover 上浮 2px + 外发光
- 底部：**重随**（灵石价）+ **跳过**（获灵石），次按钮样式
- 已拥有词条：卡片角标「已备」

### 5.3 境界突破（UI-05）

- 标题：「{境界名} · 破境」Display 金色
- 副标题：词条槽 `3 → 5` 动画计数
- 三选一天赋卡：同 AffixCard 结构，偏 **卷轴竖版**
- 确认后触发 **习得法术 Toast**（若解锁 E/R）

### 5.4 道心 Setup（UI-01）

- 背景：`bg.deep` @ 92% 遮罩
- 三个道心按钮：**问道 / 悟道 / 证道** — 横排大卡片，各带一句 flavor + 难度摘要
- 次级：**轮回成长** 入口、**踏入轮回** 主 CTA（gold 填充按钮）
- 心魔强化：独立 toggle + 碎片消耗说明

### 5.5 择路 / 坊市（UI-07 / UI-08）

- **择路：** 2–3 条路径卡片（战斗 / 坊市 / 调息），带图标与风险色边
- **坊市：** 商品网格 + 灵石余额 sticky 顶栏；购买成功短 Toast

### 5.6 随机事件（UI-06）

- 事件插画区 **16:9 占位**（后续接美术）
- 选项按钮最多 3 个，带 karma 倾向色点（善/恶/贪/逆）

### 5.7 暂停（UI-09）

- 半透明 blur（可选）+ 设置项列表
- **天象交互速查** 入口（GDD §24.4 表格 UI，P2）

### 5.8 反馈层（UI-11 / LearnToast）

| 类型 | 视觉 | 时长 | accent |
|------|------|------|--------|
| 习得法术 | 中央横卷 + 大字 | 3.2s 淡出 | spell |
| 法术换绑 | 同上，冰蓝强调 | 3.2s | rebind |
| 功法精进 | 左侧 HUD 旁弹出 | 2.5s | skill |
| 天机一击 | 全屏暗角 + 属性色 slash + 文案 | 0.3–0.8s | crit |
| 道统觉醒 | 全屏金纹 + 称号 | 1.5s | dao |
| Boss 阶段 | 顶栏 Boss 名 + 阶段名 | 2s | boss |

**伤害飘字：**

- 普通：白 `#F0ECE4`
- 天机：金 + 放大 1.2x
- 爆燃/Combo：火色 + 「!」

## 6. 组件库

### 6.1 按钮

| 变体 | 背景 | 边框 | 用途 |
|------|------|------|------|
| Primary | `accent.gold` @ 90% | 无 | 踏入轮回、确认突破 |
| Secondary | transparent | 1px `text.secondary` | 重随、跳过 |
| Danger | `#8B3A3A` | 无 | 证道难度确认 |
| Ghost | transparent | 无 | 暂停菜单项 |

**状态：** default / hover（+4% 亮度）/ pressed（scale 0.98）/ disabled（`text.muted`）  
**最小点击区域：** 44×44px（PC 也适用，便于后续手柄）

### 6.2 进度条

- **真元条：** 高 12px，`radius.sm`，背景 `#2D2D44`，填充渐变 `#E85D5D` → `#FF8A7A`
- **灵力条：** 填充 `#5B9BD5` → `#7EC8FF`
- **冷却环：** 直径 40px 环形，顺时针消退；就绪时外圈微光脉冲 1 次

### 6.3 标签 Tag

- 品质 Tag：圆角 4px，padding 2×8，文字 12px，边框品质色
- 属性 Tag：左侧 3px 色条 + 浅底 `@ 15%`

### 6.4 面板 Panel

- 最小内边距 `space.lg`
- 可选「卷轴角」装饰层（NinePatch，仅四角）
- Modal 入场：opacity 0→1，scale 0.96→1，**250ms** ease-out

## 7. 动效规范

| 场景 | 时长 | 曲线 | 备注 |
|------|------|------|------|
| 面板打开 | 250ms | ease-out | opacity + scale |
| 面板关闭 | 180ms | ease-in | 仅 opacity |
| 卡片 hover | 120ms | ease | translateY -2px |
| 习得 Toast | 320ms 入 / 900ms 出 | spring 轻 | 勿挡鼠标区域中心过久 |
| 法术栏高亮 | 1200ms | 脉冲 | 当前已实现逻辑，改为主题色动画 |
| 天机慢动作 | 300ms | — | 配合 `Engine.time_scale` |
| 词条飞入 HUD | 400ms | ease-out | 选词条后图标飞向左上 Build 区 |

**性能：** 仅动画 `modulate`、`scale`、`position`；避免每帧改 `theme_override`

**减弱动效：** 设置项「减少动效」→ Toast 改淡入、去掉慢动作（保留必要反馈）

## 8. UX 规则（优先级）

1. **可访问性：** 颜色不单独传达信息（品质需文字：凡/灵/仙…）
2. **触控/点击：** 战斗暂停时 UI 仍响应（`PROCESS_MODE_ALWAYS`）
3. **选择 > 运气：** 三选一卡片信息量对称，避免一卡明显更大
4. **发现 > 教程：** 首次天象交互用慢动作演示，不用多页说明
5. **节奏：** 清场后先 0.5s 战斗收尾，再弹词条（现有 pause 流程保持）
6. **一致性：** 所有 Modal 标题都用 Display 金色，正文 `text.primary`

## 10. Godot 实现指引

### 10.1 Theme 资源

建议新增：

```
game/ui/theme/
  samsara_theme.tres      # 主 Theme
  tokens.gd               # 颜色/间距常量（或扩展现有 game_constants）
  styles/
    panel_scroll.tres
    button_primary.tres
    progress_hp.tres
```

在 `project.godot` 设 `gui/theme/custom`，逐步替换各 Panel 的 inline `theme_override`。

### 10.2 与 EventBus 对齐

| 信号 | UI 响应 |
|------|---------|
| `learn_feedback(text, accent)` | LearnToast 样式切换 |
| `spell_unlock_changed` | 法术栏刷新 + 高亮 |
| `dao_tradition_awakened` | 全屏 overlay |
| `crit_moment_requested` | 慢动作 overlay |
| `affix_acquired` | Build 区更新 + 可选飞入动画 |
