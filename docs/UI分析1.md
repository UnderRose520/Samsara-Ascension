# 《轮回仙途》UI 调整建议——合理性评估与进一步优化

---

## 一、总体判断

这版调整建议相比之前的问题清单，在方向上有了质的提升。核心理念"无框道痕 HUD"——战斗态轻、构筑态进 Tab、主动技能少、自动系统符灯化——是正确的。它解决了之前版本"底部两行 8 按钮"和"中央常驻信息面板"的过载问题。

下面逐项评估，并指出仍需优化的地方。

---

## 二、逐项评估

### 1. "左侧管成长、右侧管战果、右上管天象、底部管主动、Tab 管解释"——合理，且清晰

这条分区原则是本版最有价值的设计决策。它用五个明确的职责边界，解决了之前"所有信息堆在右侧"和"底部 12 个按钮"的两个核心问题。

每个区域的职责单一：
- 左侧只关心"我正在变成什么"。
- 右侧只关心"我刚才做了什么"。
- 右上只关心"天地现在是什么状态"。
- 底部只关心"我现在能按什么"。
- Tab 负责所有"我想深入了解的信息"。

**合理性判断：完全合理。** 这是战斗 HUD 设计中"区域职责单一"原则的正确应用。

### 2. 底部改为三主动（Q/E/R）+ 自动符灯——合理，但有一个细节需要确认

文档要求底部中心只放 Q/E/R 三个主动技能大槽，攻/护/宠/器改为小符灯。这是对之前"12 个并列按钮"的正确精简。

**但有一个细节需要确认：** 文档说"攻/护/宠/器 自动符灯"，这意味着这四个系统默认是自动运行的。但设计手册中灵宠协同（V 键触发）和器灵觉醒（F 键触发）是手动操作，且触发时机有重要的手感锚点（灵宠协同击杀有独立的仪式级反馈）。

文档补充了"F 或 V 只在手动/满能可触发时出现"，这个临时键帽方案是合理的——平时不显示，条件满足时临时浮现，避免了常驻占用底部空间。

**但仍需优化的地方：** "临时键帽"的浮现时机和视觉设计需要更精确的规格。具体来说：

- 当器灵可以觉醒时（道势满值或特定条件满足），F 键应该从底部哪个位置浮现？从器灵符灯上方？还是从 R 槽旁边？
- 当灵宠协同可以触发时（V 键），浮现的键帽应该有什么视觉特征？它和 Q/E/R 三个常驻大槽的视觉层级如何区分？
- 如果 F 和 V 同时可用，底部会不会变成 5 个按钮（Q/E/R + F + V），又回到了按钮过多的问题？

**建议：** 临时键帽的浮现位置应该在对应的自动符灯旁边（器灵的 F 在器符灯旁边，灵宠的 V 在宠符灯旁边），而不是底部中央。这样既保持了底部中央的简洁（只有 Q/E/R），又让临时键帽的来源有归属感。

### 3. 命名统一——完全合理

文档要求"道势或道韵二选一"，这是对之前"道元 144/144""道韵 72%""道势 72/100"三个概念并存的正确修正。

**建议选择"道势"**，理由：
- "道势"在设计手册中是统一使用的术语（道势资源条、道势满值、道法通明、万法归一）。
- "道韵"更偏向"连击/节奏"的语义（设计手册中"道韵奔流"是 60 连击的里程碑名称）。
- 如果同时使用"道势"和"道韵"，玩家会混淆"道韵 72%"和"道韵奔流 ×60"的含义。

### 4. 右侧战果 rail——合理，但连击数字的显示规则需要补充

文档要求"常驻连击放右侧战果 rail，贴边显示大数字和阶段词"，"中央只在 100 连、200 连、道势满等里程碑时短暂强化"。

**合理性判断：合理。** 这解决了之前"连击在左下角太远"和"连击常驻中央侵入战斗视野"的两个问题。右侧战果 rail 贴边显示，既在视线余光范围内，又不遮挡中央战斗区。

**需要补充的细节：**

- 连击数字在右侧 rail 的字号变化规则：10 连击是小字，30 连击中字，60 连击大字，100 连击超大字——这个字号递增是否会导致右侧 rail 占用空间不稳定？如果 100 连击的数字太大，会不会侵入中央战斗区？
- 连击中断后的消失方式：是立即消失、倒数归零后消失、还是淡出？设计手册中的死亡遗憾镜头要求"连击从当前值倒数归零"，但在正常战斗中（非死亡）连击中断应该是自然淡出。

**建议：** 右侧 rail 的连击数字设定一个最大字号上限（比如不超过 rail 宽度的 2/3），超过后改为"99+"或直接用里程碑名称替代数字（如"万法将成"替代"100"）。连击中断后 1.5 秒内淡出，不倒数。

### 5. 左侧成长区——合理，但"命纹侧签"的触发和消失规则需要定义

文档提到左侧有"命纹侧签：新词条、道统推进、流派成型"和"法宝悬坠：玄玉葫 72%、器灵醒"。

**合理性判断：合理。** 左侧负责"我正在变成什么"，新词条获得、道统推进、流派成型都是成长事件，放在左侧归属清晰。

**需要补充的细节：**

- "命纹侧签"的触发条件是什么？每次获得词条都触发？还是只在关键节点（道统推进、流派成型）触发？如果每个词条都触发，左侧会频繁出现新侧签，可能干扰战斗注意力。
- 侧签的停留时间是多少？是永久显示到被新侧签替换？还是几秒后淡出？
- 如果同时获得多个成长事件（如同时获得词条和道统推进），侧签如何排列？

**建议：**
- 命纹侧签只在关键成长节点触发：道统推进（如 3/5→4/5）、流派成型（如首次触发隐藏连锁）、本命器祭炼升级。普通词条获得不在左侧触发侧签，而是在奖励选择界面中直接展示。
- 侧签停留 3 秒后淡出，但可以被玩家鼠标悬停/按键唤出查看历史。
- 同时多个侧签按时间顺序垂直排列，最多显示 2-3 条，多余的进入 Tab 历史。

### 6. 天象签采用"小签+临时展开"——合理，是最佳折中

文档说"常态：小天象签""强天气/可借势：短暂展开""详情：Tab 或 hover"。

**合理性判断：完全合理。** 这解决了之前"天象只有两个字太弱"和"天象做成大卡片太占空间"的两个问题。常态小签保持存在感，强天气时短暂展开提供战术信息，详情进 Tab 不占战斗空间。

**需要补充的细节：**

- "短暂展开"的触发条件：是"天象变化时自动展开 3 秒"？还是"玩家站在可借势地形上时展开"？还是"天象强度达到某个阈值时展开"？
- 展开后显示什么信息？只有天气名称+地形数量？还是包括"当前最佳利用方式"的提示（如"积水区域×3——雷系可导电"）？

**建议：**
- 天象签的展开规则：天象变化时自动展开 3 秒；玩家首次进入可借势地形时展开 2 秒；Boss 战开始时展开 3 秒。
- 展开内容：天气图标+名称+地形效果+借势状态（如果可用）。不做"最佳利用方式"提示——设计手册要求"借天地之势"需要玩家自己理解，不应直接教学。

### 7. Tab 玉简作为完整信息入口——合理，但需要定义 Tab 的具体内容结构

文档说"完整词条、道统缺口、宠物、法宝、统计、自动策略都进 Tab"。

**合理性判断：合理。** Tab 玉简是战斗态和构筑态之间的切换入口，所有"我想深入了解但战斗中不需要实时看到"的信息都放在这里。

**需要补充的细节：**

文档没有定义 Tab 玉简的具体内容结构和页面布局。如果 Tab 只是把所有信息平铺在一个大面板里，那它只是"把战斗 HUD 的信息移到了一个更大的面板"，并没有真正解决问题。

**建议 Tab 玉简分为以下页面：**

| 页面 | 内容 | 作用 |
|---|---|---|
| 词条页 | 当前所有已获得的词条，按道统/道脉/道胚分组 | 查看当前构筑全貌 |
| 道统页 | 道统谱系图，已获得槽位亮色、未获得暗色+类别提示 | 查看道统进度和缺口 |
| 本命器页 | 本命器当前祭炼阶段、铭纹、通灵状态 | 查看武器成长 |
| 灵宠页 | 灵宠状态、能力、协同技能 | 查看灵宠详情 |
| 法宝页 | 已装备法宝、器灵状态、觉醒进度 | 查看法宝详情 |
| 统计页 | 本局击杀数、天象击杀数、连击峰值、道势峰值等 | 查看本局数据 |
| 连锁页 | 轮回札记，已发现/未发现的隐藏连锁 | 查看连锁发现进度 |

Tab 在战斗中打开时应该暂停游戏（或大幅减速），避免玩家在查看信息时被偷袭。

## 三、仍然需要优化的四个问题

### 问题 1：左侧成长区和右侧战果 rail 的边界在实践中可能模糊

文档规定"左侧管成长、右侧管战果"，但在某些场景下，信息的归属会模糊：

- **道统推进**：当道统从 3/5 变成 4/5 时，这既是"成长事件"（左侧），也是"战斗成果"（右侧）。文档把它归入左侧"命纹侧签"，但玩家可能更习惯在右侧看到"刚才打出了一个道统推进"的即时反馈。
- **隐藏连锁发现**：触发隐藏连锁是"战斗中的操作成果"（右侧），但连锁本身是"构筑层面的发现"（左侧/Tab）。文档中隐藏连锁的"✦ 连锁发现"标签应该出现在哪个区域？
- **连击里程碑**：100 连击"万法将成"既是"战果"（右侧 rail），又有"道势+20 的成长效果"（左侧道势条变化）。

**建议：** 明确一条规则——**信息归属以"玩家第一时间想知道什么"为准，不是以"系统分类"为准。** 道统推进时，玩家第一时间想知道的是"我刚才做了什么让道统推进了"（战果视角），所以应该在右侧弹出简短提示，同时左侧的道统进度条更新。隐藏连锁发现同理——右侧弹出"✦ 连锁发现：「雷引爆」"，Tab 中的连锁页同步更新。

### 问题 2："自动符灯"的视觉语言没有定义

文档多次提到"攻/护/宠/器 自动符灯"，但没有定义符灯的视觉语言。玩家怎么通过符灯的状态判断当前自动系统在做什么？

**建议定义符灯的三种状态：**

| 状态 | 视觉表现 | 含义 |
|---|---|---|
| 常亮（低亮度） | 小图标，暗色底光 | 系统在自动运行，正常状态 |
| 脉冲（中亮度） | 小图标，呼吸灯效果 | 系统刚触发了一次动作（如灵宠刚攻击了、器灵刚产生了效果） |
| 高亮+临时键帽 | 图标亮起，旁边浮现按键提示 | 系统需要玩家手动操作（如器灵可觉醒、灵宠协同可触发） |

这样玩家通过符灯的亮度和脉冲状态，不需要看文字就知道"我的自动系统在正常运行/刚做了什么/需要我操作"。

### 问题 3：死亡遗憾镜头在新布局中的触发没有定义

设计手册中的死亡遗憾镜头是一个 3-4 秒的仪式级流程，包括时间凝固、遗憾标注、角色倒下、遗言四个阶段。但在新的 HUD 布局中，遗憾标注应该在哪个区域显示？

当前布局定义了六个区域（左上/顶中/右上/右侧/左侧/底部），但没有说明死亡镜头中这些区域如何变化。

**建议：**

- 死亡镜头触发时，所有 HUD 区域（左侧、右侧、底部）全部淡出。
- 屏幕中央成为遗憾标注的唯一展示区。
- 道势遗憾：屏幕中央显示道势条（从左上移到中央），数字跳动到满值前一格。
- 道统遗憾：屏幕中央显示道统谱系图（从左侧移到中央），未获得的槽位闪烁。
- Boss 遗憾：Boss 血条从屏幕顶部移到中央放大。
- 连击遗憾：连击数字从右侧 rail 移到中央，倒数归零。
- 遗言文字在中央停留 1.5 秒。
- 遗憾镜头结束后，所有 HUD 重新淡入，进入正常死亡结算界面。

### 问题 4：不同分辨率/屏幕比例下的自适应方案缺失

文档定义了固定的区域布局（左上/顶中/右上/右侧/左侧/底部），但没有说明在不同分辨率和屏幕比例下这些区域如何自适应。

具体来说：
- 左侧成长区和右侧战果 rail 在宽屏（21:9）上会离中央战斗区很远，玩家需要大幅度移动视线。
- 在窄屏（16:9 以下）或掌机模式下，左右两侧的区域可能互相挤压。
- 右侧战果 rail 的连击数字如果太大，可能侵入中央战斗区。

**建议：**
- 定义每个区域的最大和最小宽度（像素或百分比）。
- 在窄屏模式下，左侧成长区可以折叠为只显示道统进度条和法宝状态，隐藏词条小符印。
- 在宽屏模式下，左右两侧区域可以适当内移，靠近战斗区边缘。
- 右侧 rail 的连击数字最大字号限制为 rail 宽度的 60%，超出后自动缩小或省略。

---

## 四、总结

| 维度 | 评价 | 说明 |
|---|---|---|
| 区域职责划分 | 优秀 | 五个区域职责清晰，解决了信息过载的核心问题 |
| 底部精简 | 合理 | 三主动+自动符灯+临时键帽是正确的方向 |
| 天象签设计 | 合理 | 小签+临时展开是最佳折中 |
| Tab 玉简定位 | 合理 | 作为完整信息入口，避免战斗态过载 |
| 命名统一 | 必要 | 道势/道韵必须二选一 |
| 符灯视觉语言 | 待补充 | 需要定义常亮/脉冲/高亮三种状态 |
| 侧签触发规则 | 待补充 | 需要定义命纹侧签的触发条件和停留时间 |
| 死亡镜头适配 | 待补充 | 需要定义遗憾标注在新布局中的展示区域 |
| 分辨率自适应 | 待补充 | 需要定义各区域在不同屏幕比例下的自适应方案 |
| 信息归属边界 | 待明确 | 道统推进、连锁发现等跨区域信息的归属规则 |

**最终判断：这版调整建议的方向完全正确，"无框道痕 HUD"的理念和五区域职责划分是核心价值。剩余问题都是执行层面的细节补充，不影响整体架构。建议在 P0 阶段优先处理符灯视觉语言和底部精简，P1 阶段处理侧签规则和天象签展开逻辑，P2 阶段处理死亡镜头适配和分辨率自适应。**
