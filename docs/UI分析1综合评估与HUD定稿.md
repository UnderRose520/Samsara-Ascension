# 《轮回仙途》战斗 HUD 综合评估与定稿

更新时间：2026-06-19

## 最终判断

`docs/UI分析1.md` 的主方向合理，应采纳其核心架构：**左侧管成长，右侧管战果，右上管天象，底部管主动，Tab 管解释**。

但落地时必须继续收敛 UI 噪音。战斗 HUD 不能变成多个小卡片拼贴，也不能把所有系统都做成同级按钮。最终采用 **无框道痕 HUD**：中心 60% 留给战斗，信息贴边，常驻只保留秒读内容，长解释进入 Tab 玉简。

## 采纳内容

- 底部只保留 `Q/E/R` 三个主动大槽。
- `攻/护/宠/器` 改为自动符灯，表达后台系统状态。
- `F/V` 不常驻；只在器灵/灵宠需要手动触发时，作为小键帽浮现在对应符灯旁。
- 连击与动作反馈放右侧战果 rail，靠右贴边，不侵入中央。
- 词条、道统、流派成型、法宝成长走左侧命纹侧签。
- 天象采用右上小罗盘/天象签，常态只显示天气、地脉数量和借势状态。
- 完整词条、道统缺口、灵宠、本命器、统计、自动策略进入 Tab 玉简。
- 统一资源命名为 `道势 72 / 100`，避免道韵/道势混用。

## 不采纳内容

- 不做底部两行 8-12 个按钮。
- 不把 `Space/F/V/Tab` 都做成和 `Q/E/R` 同级的大技能槽。
- 不把连击常驻放到屏幕中央。
- 不让右侧战果流显示新词条、道统推进、构筑缺口、法宝升级。
- 不常驻显示 `缺：雷体` 这类构筑诊断，只在 Tab、奖励页或短侧签出现。
- 不使用大面积黑框、厚边框、跨屏装饰弧线、密集刻度。

## 战斗态布局

### 左上：命脉区

- 道心玉佩头像。
- 境界/身份短名。
- 命元条。
- 道势条。
- 灵石短读数。

### 左侧：成长区

- 构筑短名，例如 `雷火法修`。
- 5-6 个词条符印。
- 道统短进度，例如 `道统 · 雷罚 4/5`。
- 命纹侧签最多 2-3 条，用于 `词条入命 / 雷引 +1`、`道统推进 / 雷源 4/5`。
- 法宝悬坠显示 `玄玉葫`、`充能 72%`、`器灵醒`、`可觉醒`。
- 宠物头像作为信息入口，不显示 `V`。

### 右上：天象签

- 天气名称：`雷暴`。
- 地脉状态：`积水 x3`。
- 可借势：`借势`。
- 天气变化或进入可借势地形时短暂展开，其他时候保持小签。

### 右侧：战果 rail

- 大号连击数字和阶段词，例如 `128 / 游龙`。
- 最多 3 条动作结果：`完美身法 +1`、`火萤协同 x5`、`雷火共鸣 x3`。
- 只显示玩家刚刚打出来的结果，不显示成长信息。

### 底部：主动决策区

- `Q` 主输出。
- `E` 控场/战术。
- `R` 绝式/爆发。
- `Space` 身法作为小提示线。
- `攻/护/宠/器` 小符灯分布两侧。
- `V/F` 临时键帽只浮现在 `宠/器` 符灯旁。

## Tab 玉简

Tab 打开后暂停或弱暂停，承担完整构筑解释。

分页：

- `词条`
- `道统`
- `灵宠`
- `本命器`
- `统计`
- `自动策略`

布局：

- 左栏：词条槽位列表，按核心/临时/封印分组。
- 中央：命盘/道统图，显示当前流派、已填槽位和缺口。
- 右栏：选中项详情、来源、共鸣、本命器关系、替换/封印/分解。

## 视觉规格

- 主基调：深墨黑、玄玉绿、暗金、雷青、朱砂。
- UI 像悬浮道痕，不像矩形面板。
- 常驻状态字体不低于 12-13px；关键读数 15-18px；连击 52-72px。
- 装饰线必须有功能：进度、归属、冷却、可点击或状态变化。无功能线条全部删除。
- 宠物用圆头像/灵兽玉珠；法宝用悬坠/器物剪影；二者不能混成同一种框。

## 外部参考取法

- Hades：战斗轻 HUD、构筑详情进独立页面。

- 暖雪：国风暗黑、圣物/符印/器物感。
