# 《轮回仙途》UI分析评估与调整建议

更新时间：2026-06-19

## 总结

`docs/UI分析.md` 的问题诊断大体合理：当前战斗 HUD 的核心问题确实是信息过载、操作栏过多、天象存在感弱、战斗反馈分散、道势/道韵命名不清。

但其中部分优化建议不宜照搬。尤其是“底部两行 8 个按钮”和“连击常驻移到中央偏上”会把战斗视野重新占满，和当前已经收敛的“无框道痕 HUD”方向冲突。

推荐采用原则：

- 左侧管成长：构筑、词条、道统、法宝/器灵。
- 右侧管战果：连击、完美身法、宠物协同、元素共鸣。
- 右上管天象：天气、地脉、借势状态。
- 底部管主动：默认只放 Q/E/R 三主动。
- Tab 管解释：完整词条、道统缺口、宠物、法宝、统计、自动策略。

## 应采纳

### 1. 战斗 HUD 只留战斗必需信息

合理。角色总览、道统详情、器灵被动、词条缺口、构筑诊断不应常驻战斗 HUD。

战斗态只保留：

- 生存：命元/生命。
- 高潮资源：道势或道韵，命名必须统一。
- 目标：击杀进度、波次、倒计时。
- 可按主动：Q/E/R。
- 关键自动状态：攻/护/宠/器小符灯。
- 战果：右侧战果流。

### 2. 生命 + 道势必须一眼可读

合理。建议把“道韵 72%”统一成明确资源，例如 `道势 72/100`。如果项目决定继续叫“道韵”，也要让玩家知道它就是满值爆发资源。

道势应放在左上命脉区，与生命靠近，并和底部爆发/终式槽联动：

- 80 以上轻微脉冲。
- 95 以上加速发光。
- 满值时触发底部 R/爆发槽高亮。

### 3. 天象需要增强

合理，但不应做成常驻大说明框。

推荐右上“天象签/罗盘微章”：

- `雷暴`
- `积水 x3`
- `借势`

真正的存在感主要靠场景表现：雨丝、雷闪、地面湿痕、积水导电、天气色调。HUD 只提示当前可利用的环境要点。

### 4. 战斗反馈需要更靠近战斗感受

合理，但要分级。

- 常规反馈：`完美身法 +1`、`火萤协同 x5`、`雷火共鸣 x3` 进入右侧战果流，最多 3 条，快速淡出。
- 重大反馈：隐藏连锁发现、道势满、Boss 入场、濒死、100/200 连击里程碑，才允许中央短暂弹出。

### 5. 道统/器灵信息应简化

合理。战斗态只显示摘要，不显示完整解释。

- 道统：`道统·雷罚 4/5`。
- 法宝：`玄玉葫 72%`、`可觉醒`。
- 缺口如 `缺：雷体` 不应常驻，只在 Tab 玉简、奖励选择或短暂命纹侧签中出现。

## 不建议照搬

### 1. 底部两行 8 个按钮

不建议。它会制造新的操作过载。

`[A] 普攻 [Q] 雷引 [E] 剑阵 [R] 绝式 [Space] 身法 [V] 灵宠 [F] 器灵 [Tab] 玉简` 看似清晰，但会让所有系统都变成同级主动按钮。实际战斗中玩家会误以为每个都需要频繁操作。

特别注意：

- `A` 与 WASD 移动冲突，不应作为普攻按键。
- `Tab` 是详情入口，不应进入战斗主动栏。
- `V/F` 不应常驻大槽，除非玩家选择手动宠令/手动器灵。
- `Space` 是反应键，可做小提示或冷却线，不必和 Q/E/R 同级。

推荐方案：

```text
底部中心：Q / E / R 三主动大槽
底部两侧：攻 / 护 / 宠 / 器 自动符灯
小提示：Space 身法
临时键帽：F 或 V 只在手动/满能可触发时出现
```

### 2. 连击常驻放中央偏上

谨慎。`UI分析.md` 判断“左下连击太远”是对的，但常驻中央大连击会侵入战斗视野。

推荐：

- 常驻连击放右侧战果 rail，贴边显示大数字和阶段词。
- 中央只在 100 连、200 连、道势满等里程碑时短暂强化。

### 3. 所有战斗反馈都中央弹出

不建议。中央应保留给战斗本身和重大事件。

推荐：

- 普通战果走右侧。
- 大事件才中央弹出。
- 完美闪避用角色周围白闪、短暂停顿、右侧浮签组合表达。

### 4. 右上天象常驻三行大卡片

谨慎。极端天气可以临时强化，但常态不应变成说明面板。

推荐：

- 常态：小天象签。
- 强天气/可借势：短暂展开。
- 详情：Tab 或 hover。

### 5. 把道统/法宝状态放到底部操作栏下方

不建议。底部已经承担按键和冷却，继续塞 `道统 4/5`、`元能 72%`、`可觉醒` 会挤压主动技能可读性。

推荐：

- 道统/构筑：左侧摘要 + 命纹侧签。
- 法宝/器灵：左侧悬坠 + `器` 自动符灯。
- 详情：Tab 本命器页。

## 推荐落地布局

### 左上：命脉区

- 头像/境界。
- 命元或生命。
- 道势 72/100。
- 道势满值时与底部 R/爆发槽联动。

### 顶中：目标线

- `魔劫涌潮 · 斩魔 13/20 · 第2波 · 剩 1:51`
- 细金线 + 进度，不做任务框。

### 右上：天象签

- 天气图标 + `雷暴/晴/霜雪`。
- 地脉短提示：`积水 x3`、`无地脉`、`借势`。
- 主要天气表现交给地图与特效。

### 右侧：战果 rail

- 连击大数字。
- 阶段词。
- 1-3 条动作战果。
- 贴边、低透明竖向念珠，不向中心扩展。

### 左侧：成长区

- 构筑短名：`雷火法修`。
- 词条小符印。
- 道统短进度。
- 命纹侧签：新词条、道统推进、流派成型。
- 法宝悬坠：`玄玉葫 72%`、`器灵醒`。

### 底部：主动决策区

- Q：主输出。
- E：控场/战术。
- R：绝式/爆发。
- Space：小提示或冷却线。
- 攻/护/宠/器：小符灯，表示自动系统状态。

### Tab 玉简

完整信息都进 Tab：

- 词条。
- 道统。
- 灵宠。
- 本命器。
- 统计。
- 自动策略设置。

## 优先级

P0：

- 底部改为三主动 + 自动符灯。
- 清理重复/模糊按键：攻/普攻、护/护体、宠/协同、器灵重复。
- 命名统一：道势或道韵二选一。
- 右侧/左侧职责边界固定。

P1：

- 右上天象签增强。
- 道势满值与爆发槽联动。
- 常规战果走右侧，重大事件中央短爆点。
- 构筑缺口移入 Tab 或奖励页。

P2：

- 角色脚下辨识强化。
- 精英/Boss 标签系统。
- 天气地面覆盖和可借势提示。

## 最终判断

`UI分析.md` 是一份有价值的问题清单，但优化方案需要更新。不要采用“两行 8 按钮”和“中央常驻信息面板”的方向。最终应继续沿用“无框道痕 HUD”：战斗态轻、构筑态进 Tab、主动技能少、自动系统符灯化。
