# 战斗 HUD 参考研究：无框道痕方向

更新时间：2026-06-19

## 设计目标

《轮回仙途》的战斗 HUD 不应该再是左侧大面板堆文字。战斗中玩家最需要的是：是否会死、目标进度、能按什么、打得怎么样、资源/构筑有没有变化。其他长说明应进入暂停页、详情抽屉、奖励选择页或结算页。

推荐方向命名：**无框道痕 HUD**。

- 中心 60% 保持战斗可读，不放常驻面板。
- 右侧只承担“玩家操作成果”：连击、击杀、完美闪避、元素共鸣、宠物协同命中。新词条、道统进度、构筑成型不进右侧。
- 左上只保留生存与身份：头像/境界、命元、道韵。
- 顶部用细金线显示目标节奏，不做黑色任务框。
- 底部技能栏只放玩家当前战斗主动决策，不做系统入口清单。推荐默认 `Q/E/R` 三主动，`Space` 身法作为小提示，普攻/护体/灵宠/器灵进入自动状态符灯。
- 灵石、词条、天气、宠物变成徽章或短签，不挤成多行说明。
- 法宝/器灵是局内核心系统，要有独立入口：常驻“法宝悬坠”，主动/觉醒才进入底部 F 槽。
- 装饰线只服务定位和反馈，不能成为画面主角；删除无功能的大弧线、长竖线和密集刻度。

## 信息分层

### 常驻

- 左上：头像/道心玉佩、境界/职业短标签、命元条、道韵条。
- 顶中：`魔劫涌潮 · 斩魔 1/20 · 第2波 · 剩 1:51`，配 3-4px 目标进度金线。
- 右侧：大号连击数字、阶段词、最近 1-3 条动作战果浮签。
- 底中：`Q/E/R` 三个主动技能大槽，冷却环和按键必须一眼可读；`Space` 身法保留为小提示或冷却线。
- 底部两侧：自动系统小符灯，例如 `攻/护/宠/器`，显示自动普攻、自动护体、灵宠协同、器灵策略状态。
- 角落：灵石数字、天气符、宠物状态、词条槽位徽章。
- 左侧中段：构筑摘要与“命纹侧签”，承载新词条、道统进度、流派成型等成长提示。
- 左侧中下：法宝/器灵悬坠，显示本命器图标、器灵状态、充能/觉醒状态。

### 宠物/伙伴信息

宠物不是天气、因果或装饰信息。当前局内宠物承担四类关键功能：自动助战、`V` 协同爆发、低血护主、隐藏连锁/道势奖励。因此宠物 UI 必须贴近玩家体验，而不是藏在右上天气面板。

- **伙伴入口**：给宠物一个独立头像模块，推荐放在左下偏内侧或底部技能法轮左侧。头像可 hover/click，hover 展示短 tooltip，click 打开 `Tab` 玉简详情或宠物详情抽屉。头像只表示伙伴状态，不显示 `V` 按键。
- **操作层**：`V` 协同爆发仍放到底部技能法轮中，作为唯一的宠物主动按键槽。用火萤图标、冷却环、`V` 键位和“就绪/8s”表达可用性，避免头像和技能槽同时出现两个 `V`。
- **生存层**：护主状态靠近左上命脉区，用一个小护符/宠物珠显示 `护主` 是否本房可用。触发时左上命脉区和角色周围同时闪金。
- **战果层**：宠物命中、贯穿三敌、护主触发进入右侧动作战果浮签，例如 `火萤协同 x5`、`灵兽护主`。如果奖励的是词条、道统、道势或构筑进度，则转入左侧命纹侧签。
- **详情层**：宠物名称、被动伤害、协同伤害、冷却、元素和说明进入 `Tab` 玉简详情，不常驻长文本。

推荐布局：宠物头像模块作为“伙伴入口”，底部 `V` 槽作为“可按技能”。头像和 V 槽视觉相互呼应，但不重复长文本。左上只保留护主可用的小印，不再显示宠物名称长句。

交互建议：

- Hover 宠物头像：小浮层显示 `火萤`、`被动：自动单发助战`、`V：5 发协同爆发`、`护主：低血触发`。
- Hover/按住 V 槽：显示当前冷却、协同伤害和方向提示。
- Click 宠物头像：打开宠物详情/玉简页，展示完整属性、元素、隐藏连锁发现记录。
- 冷却完成：头像可短暂发光，底部 V 槽亮起；不再用长连线连接头像与技能槽。

### 法宝 / 器灵信息

法宝与器灵不是普通技能，也不是灵宠。它们更像“本命核心器物”：提供被动、充能、吞灵、觉醒、共鸣、护主或流派发动条件。战斗 HUD 需要让玩家知道它是否在蓄势、是否可觉醒，但不能把完整说明常驻在屏幕上。

推荐结构：**左侧法宝悬坠 + 底部 F 主动槽 + Tab 本命器详情**。

- **常驻入口**：左侧中下靠近构筑摘要与命纹侧签，做一个 36-44px 的器物悬坠图标，例如剑丸、玉葫、铜镜、雷印、血炉。不要用宠物头像框，避免误读为伙伴。
- **状态读数**：只显示 `充能72%`、`器灵醒`、`沉寂/共鸣中/可觉醒` 这类短字；充能用半环或短灵纹，不做大条。
- **主动层**：如果法宝/器灵有主动释放或觉醒，底部技能法轮右侧给 `F` 槽，标签 `器灵` 或 `觉醒`。`V` 保留给灵宠，避免按键语义冲突。
- **被动层**：如果只是自动触发，法宝悬坠常驻，触发时图标亮 0.4s，并在左侧命纹侧签显示 `玄玉葫吞灵`、`器灵共鸣：雷引 +1`。
- **战果层**：法宝造成击杀、破盾、连锁等动作结果，可以进入右侧战果流；法宝带来的词条、道统、道势、成长进度仍进入左侧命纹侧签。
- **Hover/Click**：Hover 法宝悬坠显示 2-3 行短 tooltip：名称、充能、当前触发条件。Click 打开 `Tab` 玉简的 `本命器` 页。
- **详情层**：`Tab` 玉简增加 `本命器` 页，显示法宝等级、器灵性格、充能来源、吞噬材料、觉醒条件、与词条/道统/天气的共鸣。

位置建议：

- 1280x720：悬坠落点可在左边距 44-60px、底边距 145-210px 之间，避开底部技能法轮与宠物头像。
- 1920x1080：悬坠可在左边距 64-80px、底边距 220-300px，保持贴边但不压到角色视野。
- 视觉上与宠物拉开：宠物用圆头像/生物感，法宝用悬坠/器物剪影/法轮半环。

### 主动技能与自动策略

底部技能过多会把战斗 HUD 变成键盘说明书。推荐把“主动决策”和“后台自动系统”拆开：

- **默认大槽**：只显示 `Q/E/R` 三个主动技能。它们分别承载核心术法、战术/护道技能、终式/爆发。
- **身法**：`Space` 仍是关键操作，但不做同等级大圆盘；用主技能下方的小提示或短冷却线表达。
- **普攻**：默认自动普攻或 LMB，不占大槽；用 `攻` 小符灯显示自动普攻开关。
- **护体**：默认自动护体，不占 `E` 或独立大槽；用 `护` 小符灯显示 `待命/冷却/触发`。
- **灵宠**：默认自动协同，不常驻 `V` 大槽；用 `宠` 小符灯表示自动协同。只有玩家在设置中切为手动宠令时，才临时显示 `V` 小键帽。
- **器灵/法宝**：默认自动或半自动策略，用 `器` 小符灯显示。若玩家选择手动觉醒，才显示 `F` 小键帽或临时升格为爆发槽。
- **配置入口**：自动普攻、自动护体、灵宠协同、器灵触发策略放在暂停页或 `Tab` 玉简设置，不全部展开在战斗 HUD。

推荐策略默认值：

| 系统 | 默认 | HUD 表达 |
|---|---|---|
| 普攻 | 自动开启 | `攻` 亮 |
| 护体 | 自动开启，低血/被围触发 | `护` 亮，触发时闪 |
| 灵宠 | 自动协同，精英/聚怪优先 | `宠` 亮，冷却时转暗 |
| 器灵 | 半自动或 Boss/精英优先 | `器` 半环充能 |
| 法宝爆发 | 满能保留，可选手动 | 满能时临时提示 |

### 瞬时

- 动作瞬时：连击增长、击杀、暴击、完美闪避、元素共鸣、地形击杀、宠物协同命中。
- 成长瞬时：灵石吸入、词条获得、技能升级、道统推进、流派成型。成长瞬时不进入右侧战果流。
- 濒死、受击、Boss 入场、精英出现。

## 右侧战果流修正

右侧战果流太靠近战斗中央时，会把玩家视线从角色周围拉走。调整后它只显示“刚刚由操作打出来的结果”，并尽量贴右边缘：

- 允许：`连击 128`、`游龙`、`完美身法 +1`、`火萤协同 x5`、`雷火共鸣 x3`、`破盾`、`清场`。
- 禁止：`词条入命`、`道统 4/5`、`流派成型`、`技能升级长句`、词条说明、道统缺口。
- 位置：1280x720 下主竖线建议在 `x=1110-1140`，文字向右展开；1920x1080 下建议在 `x=1660-1710`。
- 宽度：动作浮签单条不超过 150px，最多 3 条；超出时合并为 `x2/x3`。
- 动效：0.12s 弹出，0.8-1.2s 停留，向右上淡出；不向屏幕中心滑动。

### 新词条 / 道统进度

构筑变化属于“我变强了”，不是“我刚打中了什么”。推荐放在左侧构筑摘要下方，做 **命纹侧签**：

- 位置：左侧 `24-36px` 起，位于命元组与宠物头像之间，最大宽度 280px，不跨入屏幕中心 60% 战斗区。
- 形态：一条暗金/玄玉竖线 + 24px 符印 + 两行短字，不画完整大框。
- 文案：`词条入命 / 雷引 +1`、`道统推进 / 雷源 4/5`。不显示长描述。
- 动效：符印从构筑槽位附近飞入左侧竖线，金线点亮一次；停留 1.6-2.2s 后向上淡出。
- 堆叠：最多 3 条；同类连续触发合并为 `雷引 +3` 或 `雷源 4/5`。
- 关键成型：如道统觉醒、流派成型，可在左侧侧签强化金光，并在角色脚下短暂出现环形道纹，但不弹中央大窗。

### 详情页 / 抽屉

- 完整词条列表、道统缺口、流派解释、本命器详情。
- 天气说明、地形效果、种子、心魔试炼、善恶贪逆等因果统计。
- 构筑分析和局内统计。

## 视觉系统

| 形态 | 适用信息 | 表现方式 |
|---|---|---|
| 玉佩 | 头像、道心、境界 | 左上小岛，圆/水滴形，轻玉光 |
| 金线 | 目标、节奏、进度 | 顶部细线，少量刻度，不做框 |
| 法轮 | 主动技能、冷却、大招 | 底部只放 Q/E/R 三个大槽；不画跨屏大弧线 |
| 念珠/刻度 | 连击、道韵、层数 | 右侧竖向 rail，阶段越高越亮 |
| 符印 | 词条、元素、buff | 16-24px 小徽章，最多常驻 5 个 |
| 令牌/浮签 | 动作战果 | 右侧短句，2 秒内淡出 |
| 命纹侧签 | 词条、道统、构筑成长 | 左侧竖线、符印飞入、短字提示，不进入右侧 |
| 罗盘 | 天气、地形倾向 | 右上小符，不展开长描述 |
| 灵宠头像 | 宠物详情入口、伙伴存在感 | 左下/底部左侧可 hover/click 模块 |
| 自动符灯 | 自动普攻、护体、宠物、器灵 | 底部主槽两侧 16-24px 小符印，亮/暗/脉冲表达策略 |
| 灵宠玉珠 | 宠物协同、护主 | 默认小符灯 + 头像入口；只有手动宠令时显示 V |
| 法宝悬坠 | 本命器、器灵、充能、觉醒 | 左侧中下独立器物图标，半环充能，F 槽只承载主动 |

## 建议主方案

**A+B 混合：三主动 + 自动符灯 + 右侧战果浮签流**

- 右侧常驻大号 `连击 128`，下方阶段词如 `破竹 / 游龙 / 天人 / 无双`。
- 连击旁保留少量竖向念珠点，竖线透明度要低，不能比连击数字更抢眼。
- 下方浮签显示 `完美闪避 +1`、`火萤协同 x5`、`雷火共鸣 x3`，最多同时 3 条。
- 宠物的动作成果也走同一条战果流：`火萤协同 x5`、`贯穿三敌`、`灵兽护主`。
- 五行共鸣只在触发时闪现，不做复杂常驻仪表。
- 新词条、道统推进、流派成型不走右侧，统一进入左侧命纹侧签。
- 底部只保留 `Q/E/R` 三个主动大槽；普攻、护体、宠物、器灵进入 `攻/护/宠/器` 自动符灯。

## 词条与玉简构筑页

词条信息量大，不能塞进战斗 HUD 的长文本里。建议采用 **战斗摘要 + hover 快速详情 + Tab 玉简命盘** 三层。

### 战斗摘要

- 左侧中段显示当前构筑短名，例如 `雷火法修`。
- 下方显示槽位摘要：`核心3/3 临时2/2 封印1/1`。
- 再下方显示分类摘要：`术3 体1 契2`。
- 词条用 5-6 个小玉签/符印槽表示，边框表达槽位类型：核心金边，临时青玉边，封印灰紫链纹。
- 道统显示为短进度：`道统 · 雷罚 4/5`，旁边显示缺口 `缺：雷体`。
- Hover 单个词条只显示短 tooltip：名称、品质、类别、元素、一行核心效果、道统填充关系。
- 新词条与道统进度变化在构筑摘要下方以命纹侧签出现，完成“玉签归槽/道线亮起”的短动画。

### Tab 玉简命盘

按 `Tab` 打开完整构筑页，战斗暂停或弱暂停。页面结构：

- 顶部：分页 `词条 / 道统 / 灵宠 / 统计`。
- 顶部分页应扩展为 `词条 / 道统 / 灵宠 / 本命器 / 统计`。
- 左栏：核心槽、临时槽、封印槽列表，可上下选择。
- 中央：命盘/道统图，把当前构筑与道统槽位可视化。
- 右栏：选中词条详情，包括效果、来源、构筑评估、替换/封印/分解。
- 底部：快捷键提示，例如 `Esc 关闭 / Enter 详情 / X 封印 / R 替换`。

这套结构综合参考：暖雪的圣物槽位和国风器物感，Hades 的战斗轻 HUD/详情重页面，Risk of Rain 2 的图标堆叠和 hover 详情，以及幸存者类战斗中只保留秒读信息的原则。

## 落地组件建议

- 保留现有 `hud.gd` 事件接线，但把职责拆成独立组件：
  - `HudVitalsCluster`：左上生命/道韵/头像。
  - `HudObjectiveLine`：顶部目标金线。
  - `HudComboRail`：右侧连击/战果。
  - `HudPetStatus`：底部 V 宠物技能状态、左上护主小印、右侧宠物战果入口。
  - `HudArtifactPendant`：左侧法宝悬坠，显示法宝图标、器灵状态、充能/觉醒。
  - `HudBuildSummary`：战斗内构筑摘要、词条符印、道统缺口、hover tooltip。
  - `HudBuildChangeStream`：左侧命纹侧签，承载新词条、道统推进、流派成型等成长提示。
  - `HudJadeCodex`：Tab 玉简详情页，承载完整词条、道统、灵宠和统计。
  - `HudResourceBadges`：灵石、天气微章。
  - `HudSkillDock`：底部法轮技能槽。
- 当前 `HudCharacterPanel` 可逐步瘦身，不再承载所有长文本。
- 长文本迁移到暂停页或 `Tab` 详情抽屉。
