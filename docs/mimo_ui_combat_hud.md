## 二、组件设计详解

### 2.1 命脉区（HudVitalsCluster）

#### 2.1.1 玉佩头像

| 属性 | 规格 |
|------|------|
| **形态** | 圆形玉佩，直径 48px，外圈冷金细线 |
| **材质** | 深墨玉底 + 半透明白玉光晕，边缘有水墨晕染 |
| **状态** | 正常：玉光微脉冲（0.8s 周期）；低血（<30%）：玉光变红脉冲加快（0.4s）；死亡：碎裂纹理 |
| **粒子** | 正常：无；低血：玉佩周围飘散淡红色墨点；濒死：红墨点加速+屏幕边缘红晕 |

#### 2.1.2 境界/身份标签

| 属性 | 规格 |
|------|------|
| **形态** | 水平短签，宽约 120px，高 24px |
| **内容** | `{境界} · {流派}`，如 `炼气期 · 雷火法修` |
| **字体** | 13px 楷体/宋体，颜色：境界色（炼气-青，筑基-金，金丹-红，元婴-紫，渡劫-白） |
| **动画** | 突破时：文字闪烁金光 0.6s + 粒子从玉佩向外扩散 |

#### 2.1.3 真元条（命元）

| 属性 | 规格 |
|------|------|
| **形态** | 水墨笔触条，非矩形，边缘有毛笔飞白效果 |
| **尺寸** | 宽 280px，高 18px，位于玉佩下方 12px |
| **颜色** | 正常：朱砂红渐变（#8B0000 → #FF4444）从左到右 |
| **背景** | 深灰底（#1A1A1A）+ 极淡墨纹 |
| **文字** | 白色 10px 居中 `真元 92/100` |
| **状态变化** | |
| - 正常 | 朱砂红渐变，笔触边缘清晰 |
| - 低血（<30%） | 颜色变为暗红，脉冲闪烁（0.5s 周期），笔触边缘出现墨色飞溅 |
| - 受击 | 条内白色闪光 0.1s，条整体向右微震 2px |
| - 治疗 | 绿色光点从左向右流动 0.3s |
| - 死亡 | 条内红色迅速褪去，变为灰色墨痕 |

**粒子效果：**
- 低血时：条下方飘散淡红色墨点（每秒 3-5 个，向上飘 0.5s 消散）
- 治疗时：条上方飘散淡绿色光点（每秒 5-8 个，向上飘 0.8s 消散）

#### 2.1.4 道势条

| 属性 | 规格 |
|------|------|
| **形态** | 水墨笔触条，边缘有墨色晕染 |
| **尺寸** | 宽 280px，高 14px，位于真元条下方 8px |
| **颜色** | 正常：玄玉绿渐变（#004D40 → #00BFA5）从左到右 |
| **背景** | 深灰底 + 极淡墨纹 |
| **文字** | 白色 9px 居中 `道势 72/100` |
| **状态变化** | |
| - 正常 | 玄玉绿渐变，笔触边缘清晰 |
| - 满值（100%） | 金色脉冲 0.6s，条边缘发出淡金光 |
| - 消耗 | 条内白色闪光 0.1s，向左微震 2px |
| - 充能 | 青色光点从下向上流动 0.3s |

**粒子效果：**
- 满值时：条上方飘散金色墨点（每秒 2-3 个，缓慢上升）
- 消耗时：条内墨色涟漪扩散 0.2s

#### 2.1.5 灵石显示

| 属性 | 规格 |
|------|------|
| **形态** | 独立数字，非条状 |
| **位置** | 道势条下方 16px，左对齐 |
| **内容** | `灵石 💎 240`（图标 + 数字） |
| **字体** | 12px，金色（#FFD700） |
| **动画** | 获得灵石：数字 +1 滚动效果 0.2s，短暂放大 1.2x |

### 2.2 成长区（HudBuildSummary）

#### 2.2.1 构筑摘要

| 属性 | 规格 |
|------|------|
| **形态** | 垂直布局，宽 320px |
| **位置** | 左侧，命脉区下方 10px |
| **内容** | 构筑短名 + 槽位摘要 + 分类摘要 |

**示例：**
```
构筑: 雷火法修
核心3/3 临时2/2 封印1/1
术3 体1 契2
```

**字体：**
- 构筑名：14px，属性色（雷-蓝，火-橙）
- 摘要：10px，secondary 色

#### 2.2.2 词条符印槽

| 属性 | 规格 |
|------|------|
| **形态** | 5-6 个小符印，排列成水平或 2×3 网格 |
| **尺寸** | 每个 32×32px，间距 4px |
| **边框** | 核心：金边（#FFD700）；临时：青玉边（#00BFA5）；封印：灰紫链纹（#9C27B0） |
| **图标** | 五行符印（火🔥、雷⚡、水💧、木🌿、土🪨）+ 封印符 |
| **状态** | 正常：静态；新获得：从命纹侧签飞入 + 金线点亮 0.4s |

#### 2.2.3 道统进度

| 属性 | 规格 |
|------|------|
| **形态** | 短进度条 + 文字 |
| **内容** | `道统 · 雷罚 4/5` |
| **字体** | 11px，道统色（雷-蓝，火-橙） |
| **进度条** | 宽 160px，高 6px，属性色渐变 |
| **动画** | 进度增加：条内光点流动 0.3s |

#### 2.2.4 命纹侧签

| 属性 | 规格 |
|------|------|
| **形态** | 垂直竖线 + 符印 + 短文字 |
| **位置** | 左侧成长区内，靠近左边缘 24px |
| **尺寸** | 宽 248px，最大高度 180px |
| **内容格式** | `{符印图标} {成长类型} / {具体变化}` |
| **示例** | `⚡ 词条入命 / 雷引 +1`、`📜 道统推进 / 雷源 4/5` |

**视觉规格：**
- 竖线：2px 宽，暗金色（#B8860B），半透明
- 符印：24×24px，从构筑槽位附近飞入
- 文字：10px，secondary 色
- 动画：符印飞入 0.3s → 金线点亮 0.4s → 停留 1.6s → 向上淡出 0.4s
- 堆叠：最多 3 条；同类合并（如 `雷引 +3`）

### 2.3 天象签（HudWeatherCompass）

#### 2.3.1 形态与位置

| 属性 | 规格 |
|------|------|
| **形态** | 小型玉质罗盘/签牌，非大卡片 |
| **尺寸** | 160×96px |
| **位置** | 右上角，距顶 0px，距右 0px |
| **背景** | 半透明深墨底 + 淡金边 |

#### 2.3.2 内容

```
┌──────────────────────┐
│  [罗盘图标] 32×32    │
│  雷暴 · 积水 x3      │
│  可借势              │
└──────────────────────┘
```

**字体：**
- 天气名：16px，天气色（雷-蓝，火-橙，雨-灰蓝，雪-银白，风-黄褐）
- 地形/借势：10px，secondary 色

#### 2.3.3 状态变化

| 状态 | 视觉变化 |
|------|----------|
| 正常 | 罗盘图标缓慢旋转（360°/10s） |
| 可借势 | 罗盘图标发出对应属性色光晕，文字闪烁 0.6s |
| 天气变化 | 罗盘图标快速旋转 1 圈 0.5s + 属性色粒子扩散 |

**粒子效果：**
- 可借势时：罗盘周围飘散对应属性色墨点（如雷暴-蓝色电弧点）
- 天气变化时：从罗盘向外扩散属性色冲击波 0.3s

### 2.4 连击/战果区（HudComboRail）

#### 2.4.1 连击数字

| 属性 | 规格 |
|------|------|
| **形态** | 大号数字 + 阶段词 |
| **位置** | 右侧，距右边缘 0px，垂直居中偏上 |
| **尺寸** | 数字 72px，阶段词 18px |
| **颜色** | 基础：白色；阶段越高越亮（10连-白，100连-金，200连-白金发光） |

**阶段词：**
```
0-9:    （无）
10-49:  破竹
50-99:  游龙
100-199: 天人
200+:   无双
```

#### 2.4.2 战果浮签

| 属性 | 规格 |
|------|------|
| **形态** | 短句浮签，最多 3 条 |
| **尺寸** | 单条宽 ≤150px，高 24px |
| **位置** | 连击数字下方，垂直排列，间距 8px |
| **内容** | 操作成果：`完美身法 +1`、`火萤协同 x5`、`雷火共鸣 x3` |
| **字体** | 11px，战果色（完美-金，协同-属性色，共鸣-白金） |

**动画：**
- 入场：0.12s 从右侧弹入
- 停留：0.8-1.2s
- 退场：向右上淡出 0.3s
- 不向屏幕中心滑动

**粒子效果：**
- 完美闪避：白色墨点从角色向右飞溅 0.2s
- 元素共鸣：对应属性色墨点从技能命中点向外扩散

#### 2.4.3 念珠竖线（可选装饰）

| 属性 | 规格 |
|------|------|
| **形态** | 连击数字左侧的垂直念珠点 |
| **尺寸** | 点直径 4px，间距 8px，最多 5 点 |
| **颜色** | 透明度 30%，不比连击数字更抢眼 |
| **用途** | 表示连击阶段（每 50 连亮一点） |

## 十二、总结

本设计方案遵循「无框道痕 HUD」核心理念：

1. **中心 60% 留给战斗** — 所有信息贴边，不侵入中央
2. **信息分层清晰** — P0 生死秒读 → P1 节奏反馈 → P2 构筑感知 → P3 深度详情
3. **水墨美学统一** — 所有元素使用墨笔触、飞白边缘、晕染效果
4. **粒子效果克制** — 只在关键反馈时启用，不常驻干扰
5. **响应式适配** — 锚点策略覆盖 16:9、16:10、21:9、4:3
6. **无障碍友好** — 色彩对比度达标，色盲友好图标，屏幕阅读器支持
