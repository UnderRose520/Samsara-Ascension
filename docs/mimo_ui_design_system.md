# 《轮回仙途》设计系统基础

> 版本：1.0  
> 引擎：Godot 4.6 · 2D 顶视角  
> 基准分辨率：1920×1080  
> 视觉基调：水墨暗色 × 高饱和粒子光效  

---

## 1. 色彩体系

### 1.1 基础色阶（墨色体系）

从纯黑到宣纸白的完整灰阶，带暖调偏移，避免冷灰破坏水墨感。

| Token | 名称 | HEX | RGB | 用途 |
|---|---|---|---|---|
| `ink.000` | 纯墨 | `#0A0A0F` | (10,10,15) | 最深背景、游戏空间底色 |
| `ink.100` | 浓墨 | `#121218` | (18,18,24) | 面板底层、遮罩层 |
| `ink.200` | 重墨 | `#1A1A22` | (26,26,34) | 卡片/面板背景 |
| `ink.300` | 深墨 | `#222230` | (34,34,48) | 次级面板、分隔区域 |
| `ink.400` | 中墨 | `#2E2E3E` | (46,46,62) | 悬浮状态、侧边栏 |
| `ink.500` | 淡墨 | `#3E3E52` | (62,62,82) | 边框、分隔线 |
| `ink.600` | 清墨 | `#5A5A72` | (90,90,114) | 次级文字、占位符 |
| `ink.700` | 灰墨 | `#787890` | (120,120,144) | 辅助说明文字 |
| `ink.800` | 宣白 | `#B8B0A8` | (184,176,168) | 主体文字（暖白） |
| `ink.900` | 素纸 | `#E8DFD2` | (232,223,210) | 标题文字、高亮文本 |
| `ink.950` | 净纸 | `#F5EEE2` | (245,238,226) | 最高亮文本、纯白替代 |

**暖调偏移原则**：所有灰阶加入微量暖黄（R 通道 +2~5），模拟宣纸质感，避免冷灰破坏水墨氛围。

### 1.2 元素色（五行+混沌）

五行元素色模拟"墨入水中"的扩散效果，每个元素有基础色和高亮色两档。

| 元素 | 基础色 | HEX | 高亮色 | HEX | 视觉描述 |
|---|---|---|---|---|---|
| 🔥 火 | 朱砂暗 | `#8B2500` | 朱砂明 | `#FF4400` | 暗沉如干涸血迹，亮时如熔岩流淌 |
| 💧 水 | 玄青暗 | `#1A3A4A` | 玄青明 | `#00CCFF` | 深潭般沉静，亮时如月光碎水面 |
| ⚡ 雷 | 紫金暗 | `#3A1A5A` | 紫金明 | `#BB44FF` | 暗处如雷云底部，亮时如闪电撕裂 |
| 🌿 木 | 青碧暗 | `#1A3A1A` | 青碧明 | `#44FF44` | 深林般幽暗，亮时如萤火丛中 |
| 🪨 土 | 赭石暗 | `#4A3A1A` | 赤金明 | `#FFAA22` | 泥土般厚重，亮时如黄昏金光 |
| 🌀 混沌 | 墨渊暗 | `#1A1A2A` | 虹彩明 | `#FF44FF` | 不可名状的深渊色，亮时如万花筒碎裂 |

**元素色使用规则**：
- 基础色用于：被动状态、未激活技能、背景氛围渲染
- 高亮色用于：技能释放特效、粒子爆发、主动状态图标
- 两种颜色在同一画面中比例不超过 3:7（基础色为主）

### 1.3 品质色阶（凡/灵/仙/天/道）

品质色随稀有度递增，亮度和饱和度同步攀升，形成"道行递进"的视觉叙事。

| 品级 | 中文名 | HEX | 饱和度 | 亮度 | 附加效果 |
|---|---|---|---|---|---|
| 凡品 | 凡 | `#888888` | 0% | 53% | 无光效，朴素灰白 |
| 灵品 | 灵 | `#44CCAA` | 70% | 60% | 微弱内发光，呼吸闪烁 |
| 仙品 | 仙 | `#4488FF` | 85% | 70% | 中等外发光，边缘流光 |
| 天品 | 天 | `#AA44FF` | 90% | 80% | 强烈光环，粒子环绕 |
| 道品 | 道 | `#FFD700` | 95% | 90% | 帝王金光，全屏震动反馈 |

**品质色叠加规则**：高品质文字在暗底上必须带发光投影，确保可读性。道品文字额外加 2px 动态模糊光晕。

### 1.4 语义色

| 语义 | HEX | 用途 |
|---|---|---|
| 生命值（HP） | `#FF3344` | 生命条、治疗数字、回血粒子 |
| 法力值（MP） | `#3366FF` | 法力条、法术消耗数字 |
| 精神值（SP） | `#AA44FF` | 精神条、心魔相关 UI |

| 伤害数字（物理） | `#FFFFFF` | 白色弹出伤害 |
| 伤害数字（元素） | 对应元素高亮色 | 元素伤害弹出 |
| 暴击数字 | `#FFD700` | 暴击伤害特殊弹出 |
| 治疗数字 | `#44FF88` | 回血数字 |
| Buff 增益 | `#44DDFF` | 增益状态图标边框 |
| Debuff 减益 | `#FF4466` | 减益状态图标边框 |
| 领悟/学习 | `#FFEE44` | 技能领悟、心法参悟 |
| 仇恨/危险 | `#FF0000` | 敌意指示、危险区域 |
| 安全/安全区 | `#44BB44` | 安全区域指示 |

### 1.5 色彩比例指导

在任意 UI 界面中，颜色面积占比遵循：

```
墨色系（ink.000-ink.500）  ：60%   — 背景、面板、空间
中性色（ink.600-ink.950）  ：25%   — 文字、线条、图标
元素色/品质色              ：10%   — 强调、焦点、状态
语义色                     ： 5%   — 功能指示、反馈
```

### 2.4 特殊字体规则

**伤害数字**：
- 基础伤害：`JetBrains Mono Bold 32px`，白色 `#FFFFFF`，描边 2px 黑色
- 暴击伤害：`JetBrains Mono Black 48px`，金色 `#FFD700`，描边 2px + 外发光 4px
- 治疗数字：`JetBrains Mono Bold 32px`，绿色 `#44FF88`
- 弹出动画：Y 轴上移 40px + 淡出，持续 0.8s，`easeOutCubic`

**境界名称**：
- 使用展示字体，48px 以上
- 凡/灵/仙/天/道 各品质加对应品质色发光
- 突破时文字从 72px 缩放到 48px + 金色粒子爆发

## 3. 间距与布局网格

### 3.1 基础网格

以 **8px** 为基本单位，所有间距、尺寸为 8 的倍数。

| Token | 值 | 用途 |
|---|---|---|
| `space.0` | 0px | — |
| `space.1` | 4px | 微间距（图标与文字间隙） |
| `space.2` | 8px | 紧凑间距（行内元素间隙） |
| `space.3` | 12px | 默认间距（列表项间距） |
| `space.4` | 16px | 标准间距（段落间、按钮内边距） |
| `space.5` | 24px | 宽松间距（面板内边距） |
| `space.6` | 32px | 大间距（面板间、区域分隔） |
| `space.7` | 48px | 超大间距（页面分区） |
| `space.8` | 64px | 特大间距（全屏布局锚点） |
| `space.9` | 96px | 留白（成就/突破全屏画面） |

### 3.2 圆角系统

水墨风格以锐角和不规则边缘为主，圆角极少使用。

| Token | 值 | 用途 |
|---|---|---|
| `radius.none` | 0px | 默认面板（锋利边缘） |
| `radius.sm` | 2px | 按钮、小元素（微圆） |
| `radius.md` | 4px | 输入框、下拉菜单 |
| `radius.lg` | 8px | 弹窗、模态框（仅特殊场景） |
| `radius.circle` | 9999px | 圆形按钮、头像裁切 |

**水墨边缘处理**：重要面板不用圆角，改用 `Shader` 模拟墨迹边缘——通过 noise texture 做 alpha clip，让矩形面板边缘呈现毛笔飞白效果。

### 3.3 阴影系统（墨晕投影）

阴影模拟墨滴在宣纸上的自然晕染，带有暖色偏移。

| Token | 偏移 | 模糊 | 颜色 | 用途 |

| `shadow.xs` | 0px 2px | 4px | `rgba(10,10,15,0.3)` | 按钮微浮 |
| `shadow.sm` | 0px 4px | 8px | `rgba(10,10,15,0.4)` | 卡片悬浮 |
| `shadow.md` | 0px 8px | 16px | `rgba(10,10,15,0.5)` | 面板投影 |
| `shadow.lg` | 0px 12px | 24px | `rgba(10,10,15,0.6)` | 弹窗投影 |
| `shadow.xl` | 0px 16px | 32px | `rgba(10,10,15,0.7)` | 全屏遮罩层 |

**墨晕特殊投影**：用于 Boss 出场、境界突破等重要场景。
- 颜色：`rgba(255,215,0,0.15)`（淡金色晕染）
- 偏移：0 0
- 模糊：48px~96px（呼吸动画扩展）

## 4. 图标系统

### 4.1 图标风格

**水墨笔触图标**：
- 线条：2~3px 粗细，带有毛笔起笔/收笔的粗细变化
- 颜色：默认使用 `ink.800`（暖白），激活态使用对应元素色
- 不使用填充色，纯线条勾勒（与游戏暗底形成对比）
- 笔触末端：略带飞白效果（透明度渐变到 0）

### 4.2 图标尺寸

| Token | 尺寸 | 用途 |
|---|---|---|
| `icon.xs` | 16×16 | 状态指示小圆点 |
| `icon.sm` | 24×24 | 列表内联图标 |
| `icon.md` | 32×32 | 技能栏图标、装备图标 |
| `icon.lg` | 48×48 | 面板标题装饰、Boss 标记 |
| `icon.xl` | 64×64 | 境界突破图标、成就图标 |

### 4.3 图标动画原则

| 动画 | 参数 | 用途 |
|---|---|---|
| 呼吸闪烁 | 缩放 1.0↔1.05，2s 周期，easeInOutSine | 激活技能、可交互元素 |
| 墨迹扩散 | 从中心向外 alpha 淡入，0.3s，easeOutCubic | 图标出现 |
| 笔触勾勒 | 按笔画顺序 reveal，0.6s | 加载动画、新技能解锁 |
| 震动闪烁 | X/Y ±2px 随机偏移，0.1s | 受击反馈、技能可用 |

### 5.1 面板背景

|---|---|---|
| `bg.solid` | 纯墨色 `ink.200` (#1A1A22) | 默认面板背景 |

| `bg.cloud` | 流动云纹 shader（半透明白色在 `ink.000` 上） | 加载界面、仙境场景 |

**宣纸纹理规格**：
- 尺寸：256×256，平铺
- 颜色：`ink.800` 线条在 `ink.100` 底色上
- 透明度：15%~25% 叠加到面板背景
- Godot 实现：`CanvasItem` + `TileMap` 或自定义 `Shader`

### 5.2 边框样式

| Token | 描述 | 用途 |

| `border.brush` | 2~3px 笔触线条（左右粗中间细） | 重要面板边框 |
| `border.glow` | 1px 元素色 + 外发光 4px | 品质边框、选中态 |
| `border.seal` | 印章边框（方形+篆刻边缘） | 成就、里程碑 |

### 5.3 发光/霓虹系统

发光效果用于：品质高亮、技能激活、粒子光源。

| Token | 模糊半径 | 颜色 | 用途 |
|---|---|---|---|
| `glow.subtle` | 4px | 元素基础色 50% | 灵品装备边框 |
| `glow.medium` | 8px | 元素高亮色 60% | 仙品技能激活 |
| `glow.strong` | 16px | 元素高亮色 80% | 天品光环 |
| `glow.epic` | 32px | 道品金色 100% | 道品突破、终极技能 |

**发光叠加规则**：
- 发光层始终在内容层之下、背景层之上
- 多个发光重叠时使用 `Add` 混合模式
- 发光强度随帧率波动 ±10%，模拟呼吸感

## 7. 设计原则

### 7.1 五大核心视觉原则

**一、墨分五色——层次即叙事**

水墨画讲究"焦、浓、重、淡、清"五墨，游戏 UI 也应通过墨色深浅建立信息层级。最重要的信息最亮（接近宣纸白），背景最深（接近纯墨），中间层次用灰阶过渡。绝不允许两个同亮度元素相邻。

**二、计白当黑——留白即力量**

水墨画的留白不是空白，是"气"的流动空间。UI 面板之间、元素之间必须留出呼吸间距。宁可信息少一些，也不可拥挤。密集的信息用滚动/分页处理，而非压缩间距。

**三、以黑衬彩——克制即是张力**

全局 85% 以上的视觉面积应为墨色系。高饱和的元素色、品质色是"点睛之笔"，必须极度克制使用。当画面中只有一处亮色时，它就是绝对焦点。满屏彩色 = 无彩色。

**四、动静相生——静中有动**

静态 UI 应有微动效（呼吸、闪烁、流动），让界面"活着"。但动效必须克制——一个界面同时运动的元素不超过 3 个。大面积的运动用于重要时刻（突破、Boss 出场），日常保持静谧。

**五、笔断意连——断裂中见统一**

水墨画的笔触常有断裂，但气韵连贯。UI 元素之间不需要物理连接线，通过间距、对齐、色调统一建立视觉关联。分隔线尽量用"墨迹间隙"代替实线。

### 7.2 设计禁忌

| 禁忌 | 原因 | 替代方案 |
|---|---|---|
| ❌ 使用纯白 `#FFFFFF` 作为大面积背景 | 破坏暗色基调，刺眼 | 使用 `ink.950`（#F5EEE2）宣纸白 |
| ❌ 多种高饱和色同时大面积出现 | 视觉噪音，失去焦点 | 同一界面最多 2 种元素色，面积 <15% |
| ❌ 使用标准圆角矩形按钮 | 与水墨风格冲突 | 使用笔触边框 + 不规则边缘 shader |
| ❌ 添加写实渐变（金属/玻璃） | 风格不统一 | 用墨色渐变 + 发光替代 |
| ❌ 使用硬阴影（sharp drop shadow） | 工业感过强 | 使用墨晕投影（模糊 + 暖色偏移） |
| ❌ 动画节奏均匀 | 失去水墨"疾徐有致"的韵律 | 使用自定义缓动曲线，快慢交替 |
| ❌ 过度使用描边文字 | 显得廉价 | 用发光投影替代描边 |
| ❌ 使用像素字体/科技字体 | 与修仙主题冲突 | 使用宋体/手写体/书法体 |

### 7.3 可访问性补充

- 所有文字与背景对比度 ≥ 4.5:1（WCAG AA）
- 关键状态（HP 低、Boss 警告）不仅依赖颜色，同时有图标/文字/动画辅助
- 色盲友好：元素色除色相外，同时用形状/图标区分（火=尖角、水=波浪、雷=锯齿等）
- 提供"高对比度模式"选项：所有文字加粗 + 纯白，面板加实色边框
